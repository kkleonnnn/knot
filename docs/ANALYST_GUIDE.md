# KNOT 业务方使用指南（analyst onboarding）

> 面向：运营 / 业务 / 数据分析人员（analyst 角色）。
> 你不需要懂 SQL —— 用中文提问，KNOT 自动生成 SQL、图表和洞察。
> 配套：[管理员指南](ADMIN_GUIDE.md) · [文档导航](ONBOARDING.md)

---

## 1. KNOT 是什么

KNOT 是一个 **自然语言 BI 助手**：你用日常语言问数据问题，它把问题翻译成只读查询，跑在业务数据库上，再用图表 + 文字洞察把结果讲清楚。

当前阶段定位 = **"更好的 ChatBI"**（团队内部工具，日活 5-20 人）。能力边界见 §6 —— 请先读一遍，避免问超出范围的问题白等。

---

## 2. 第一次提问

进入对话页，直接输入问题，回车发送（Shift+Enter 换行）。

**好问题的特征**：
- **说清主体 + 指标 + 时间**：「最近 7 天每天的下单用户数」比「看看用户」清楚得多
- **一次一个意图**：先问总量，再追问明细，比一句话塞 3 个问题更稳
- **用业务词**：如果管理员配过业务词典（如「GMV」「活跃用户」），直接用这些词命中更准

**示例**：
- 「上个月各渠道的销售额排名」
- 「本周每天的新增用户数趋势」
- 「对比这个季度和去年同期的订单量」（同比）

如果问题不够清楚，KNOT 会 **反问澄清**（Clarifier step）—— 补充一句即可，不用重头再来。

## 3. 读懂结果

### 4-step 思考过程（透明可追溯）
每次查询右侧会展示 4 步：

| 步 | 名 | 在做什么 |
|---|---|---|
| K | Knowledge | 理解你的问题（口径 / 时间 / 主体）|
| N | Nexus | 生成 SQL（ReAct 推理，含表关系约束）|
| O | Objective | 给出洞察 + 异常提示 + 追问建议 |
| T | Trace | 推导信任度（用了几张表 / 几条 SQL / JOIN / 聚合 → 高/中/低）|

**信任度低**时多留个心眼，建议核对 SQL 或换个问法。

### 结果呈现（7 种）
KNOT 按问题类型自动选呈现方式：指标卡（单值）/ 折线（趋势）/ 柱状（对比）/ 饼图（占比）/ 排行榜（TopN）/ 留存矩阵 / 明细表。每个结果都可展开看生成的 SQL。

---

## 4. 多轮追问

KNOT 记得上下文 —— 代词会自动回指上一题：
- 你：「上月各城市订单量」→ 「**这些城市**里哪些环比下降了」（自动继承「上月 + 城市」口径）

追问继承上一题的口径，不用每次重复时间范围。

---

## 5. 收藏 / 重跑 / 导出

- **收藏**：满意的结果点书签，进「收藏报表」屏长期留存
- **重跑**：收藏的报表可一键用最新数据重跑
- **导出**：结果支持 CSV / Excel(xlsx) 导出（大结果有行数上限保护）
- **追问继承**：收藏报表保留原始问题，方便回溯口径

## 6. 能做 / 不能做（请务必读）

> 与 README「knot 当前能做什么 / 不能做什么」字面一致（v0.6.1 R-PA-PB-1 窄场景宣告）。

### ✅ 当前能做（v0.6.x）
- 自然语言 → 只读 SELECT：单表 / 多表 JOIN / GROUP BY / 时间过滤 / 排序 / TopN / 子查询 / CTE
- 简单业务问题：销售统计 / 用户行为 / 趋势 / 维度对比 / 同比环比
- 可视化：折线 / 柱 / 饼 / 表 / 指标卡 / 排行 / 留存
- 时间语义：5 类核心时间表达 + 同比基准 + 节假日上下文 + D-1 数据延迟
- 业务流：收藏 + CSV/xlsx 导出 + 重跑 + 多轮追问继承

### ❌ 当前不能做（推迟 v0.7+）
- **归因分析**（"为什么下降"）—— 事件 + 规则层未建
- **跨业务域聚合**（不同业务目录 catalog 跨域分析）—— 对象语义层未建
- **数据合理性反检**（"这个数对吗"）—— 校验段未建
- **动作触发**（通知 / 工单 / 审批）—— 动作语义层未建
- **多租户隔离 / SSO / 权限分级（RBAC）**
- 一次只能基于 **当前选定的一个业务目录** 查询（不跨目录联合分析）

遇到 ❌ 范畴的问题，KNOT 会礼貌拒识并说明原因 —— 这是设计如此，不是 bug。

## 7. FAQ / 常见情况

**Q：结果好像不对 / 数字怪？**
A：展开看生成的 SQL 核对口径；若信任度标「低」，换更明确的问法（点明时间 + 主体 + 指标）。仍不对 → 反馈给管理员，可能需要补业务词典 / few-shot 示例。

**Q：为什么我的问题被拒识了？**
A：多半落在 §6 ❌ 范畴（归因 / 跨域 / 反检 / 动作）。这些是 v0.7+ 才有的能力。

**Q：查询很慢？**
A：复杂多表 + 大时间范围会慢；缩小时间范围或先问聚合再问明细。

**Q：「昨天」会不会被理解成模型训练时的日期？**
A：不会。KNOT 用 Asia/Shanghai 时区把「昨天 / 最近 7 天 / 上月」换算成绝对日期再查。

**Q：数据是实时的吗？**
A：默认按 D-1（前一天）口径，存在数据更新延迟；具体以你们的数仓更新节奏为准。
