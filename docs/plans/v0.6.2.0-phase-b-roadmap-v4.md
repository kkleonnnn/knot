### 0.3 KNOT 架构明示（v4 新增 — Stage 2 II.1 防同误判）

**两类数据库严格分离**：
- **KNOT 元数据**：`knot/data/knot.db` (SQLite) — 用户 / 会话 / 消息 / 知识库 / 上传文件 / audit_log / **TOTP secret + recovery codes（v0.6.2.0+）** / catalog seed / fewshot / prompts
- **业务查询目标**：Doris / MySQL — 用户提问 → KNOT 生成 SQL → 查业务库；KNOT 不写业务库；KNOT 不在业务库存元数据

**事务边界**：SQLite 单库原生 ACID；业务库读取 only；**严禁跨 SQLite + Doris 全局事务**。

R-46-Tx / R-PB-A1-5 / audit_log 全部限定 SQLite 范围。
