- `users.totp_enrolled_at` / `totp_last_used_at`
- `totp_recovery_codes` 表（secret_hash + used_at；单次使用）

R-46-Tx 事务模式适用范围限定 SQLite 单库。

## §5 验收标准

### 5.1 功能验收

- [ ] 已 enroll 用户 login → 返 interim_token → /api/totp/verify → 完整 JWT
- [ ] recovery code 单次使用 + 用完 audit_log INSERT
- [ ] admin 重置 user TOTP → audit_log INSERT + 该用户下次 login 重新 enroll
- [ ] 月 ≥ 5 次重置 → admin 警报（mock 集成测）

### 5.2 红线守护测试（R-PB-B1-1~11 全覆盖）
- [ ] **R-PB-B1-9 R-46-Tx**：mock `totp_repo.insert_recovery_codes` 失败 → `users.totp_secret` 未持久化（事务回滚）
- [ ] **R-PB-B1-10 race**：`asyncio.gather(*[verify(user_id, "wrong") for _ in range(100)])` 误差 ≤ 1（rate_limit 命中 95/100 ± 1）
- [ ] **R-PB-B1-6 rate limit**：单 user 5/min 第 6 次返 429
- [ ] **R-PB-B1-7 enroll 二次确认**：第二个动态码错误 → 拒绝 + secret 未持久化
- [ ] **R-PB-B1-1/8 Fernet**：sqlite3 直读 `users.totp_secret` 必含 `enc_v1:` 前缀；解密返原始 32 字符 base32 secret

## §7 风险与依赖

| 风险 | 影响 | 缓解 |
|---|---|---|
| pyotp / qrcode 依赖引入新维护面 | 安全更新跟进 | 用业界主流（pyotp 2014+ / qrcode 2011+）— RFC 6238 标准 |
| admin 自身 enroll 锁死 | 公测无法启动 | R-PB-B1-3 启动期 24h 宽限 + R-PB-B1-7 二次确认防扫码错误 |
| R-46-Tx SQLite 事务在大并发下性能 | 写入延迟 | enroll 是低频操作（用户一生 1-N 次）— 不优化 |
| race 100× rate limit 误差 ≤ 1 | 并发场景泄漏 | R-PB-B1-10 强守护测试套 |
| Fernet master_key 丢失 | TOTP secret 永久无法解密 | 同 v0.4.5 R-46-Tx 现状 sustained（master_key 是项目根本依赖）|
| 前端 React 19 + qrcode.react 兼容性 | UI 渲染失败 | 用 inline SVG 备选；不依赖 npm package（开源 readiness 加分）|
| 业务方耐心被 enroll 流程消耗 | 上线节奏 | R-PB-GOV-1 按节奏 + recovery codes 体验友好 |

### 撤回机制
- Stage 2/3 期间发现红线冲突无法妥协 → 推迟 v0.6.2.1 或大改方案
- 业务方不需要强制 → 撤回 OOS（保留 enroll 选项但 admin 可豁免）
