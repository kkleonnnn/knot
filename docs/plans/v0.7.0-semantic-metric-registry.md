## §2 指标数据模型（`metrics` 表 schema · R-SL-2）

承 prestudy §2.1 设想 + catalogs 表先例（schema.sql:159-169）+ OOS-1 死线。**字段集待 Stage 2/3 byte-equal LOCKED**：

```sql
-- v0.7.0 语义层第一刀：指标注册表（单一真源指标定义）
-- ⚠️ OOS-1 死线 sustained：metric 归 catalog_id（语义层水平切分）；严禁 tenant_id / project_id 列。
-- catalog_id = 逻辑外键（soft ref）→ catalogs(id)；无 enforced FOREIGN KEY 子句
--   （与先例 users.active_catalog_id / audit_log.catalog_id 同为裸 INTEGER；项目未启 PRAGMA foreign_keys）。
CREATE TABLE IF NOT EXISTS metrics (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    catalog_id      INTEGER NOT NULL DEFAULT 1,   -- soft ref catalogs(id)；OOS-1：水平切分非租户
    name            TEXT    NOT NULL,             -- 唯一标识（per catalog_id 唯一）
    display         TEXT    DEFAULT '',           -- 中文展示名
    aliases         TEXT    DEFAULT '',           -- JSON list（v0.7.1+ 喂 LEXICON / clarifier）
    caliber         TEXT    NOT NULL,             -- 口径表达式 SUM(o.pay_amount)（非密 — §3.7）
    base_object     TEXT    DEFAULT '',           -- 挂的对象/表（v0.7.2 对象层消费）
    filters         TEXT    DEFAULT '',           -- JSON list 口径内置过滤（非密）
    dimensions      TEXT    DEFAULT '',           -- JSON list 可下钻维度（v0.7.1+ 消费）
    lineage         TEXT    DEFAULT '',           -- JSON list 派生依赖（inert — v0.7.1 编译时校验）
    freshness_lag_days INTEGER DEFAULT 1,         -- 复用 time_resolver D-1 默认
    enabled         INTEGER DEFAULT 1,            -- 软开关
    created_at      TEXT    DEFAULT (datetime('now','localtime')),
    updated_at      TEXT    DEFAULT (datetime('now','localtime')),
    UNIQUE (catalog_id, name)                     -- per-catalog 指标名唯一
);
CREATE INDEX IF NOT EXISTS idx_metrics_catalog ON metrics(catalog_id);
```

**migration**：base.py:init_db 模式 —— `conn.executescript(_SCHEMA_SQL)`（base.py:28）幂等 CREATE TABLE IF NOT EXISTS；纯新表无 ALTER。seed 模式参考 catalogs seed（base.py:146）。**0 seed**（fresh install 空指标；kk 按查询热度增量定义 — [[kk-sole-data-analyst]]）。

**lineage 字段（对抗自查修 — 解 unsound guard + OCCAM 一致性）**：v0.7.0 lineage 仅 **inert 存储**（opaque JSON，**无 v0.7.0 消费者** — 编译留 v0.7.1）。**不在 v0.7.0 建校验器**：① 真循环检测（自引用 + 传递 A→B→A）须 **DFS visited-set**，**非** audit_service `_scrub` 的 depth-truncator（后者 `_MAX_DEPTH=3` 只截断、无 cycle 检测 — 初稿误用）；② 与判定 A 一致（无消费者代码 = 违奥卡姆）。→ lineage 自引用/循环/深度校验**随派生编译一并留 v0.7.1**（届时 DFS 真守护）。
