## §2 红线（本 PATCH 不可违反）

| ID | 红线 |
|---|---|
| **R-脱敏-1** | 表名替换必须 **word-boundary** (regex `\b` 严格) — 防 `user` 误替换业务无关 `username` 等 |
| **R-脱敏-2** | **fail-open** — lexicon 缺失或 alias_map 空时**不替换**（不阻塞业务，保持原文）|
| **R-脱敏-3** | **admin 路径 0 改动** — 已有 v0.6.0.17 守护 (sql_text strip)；此 PATCH 仅扩展到 explanation/db_error，admin 仍看完整原文 |
| **R-脱敏-4** | **不替换 SQL 字符串内部** — sql_text 已被 v0.6.0.17 strip；本 PATCH 仅扫 explanation/db_error 两字段 |
| **R-脱敏-5** | **字段名暂不脱敏** — 字段太多，误替换风险高；留待 v0.6.0.x / v0.6.2.x 评估（如有真实需求拉动）|
| **R-脱敏-6** | **case insensitive 匹配 + case preserving 替换** — SQL 通常 case insensitive，但保留替换前后 case；使用 `re.IGNORECASE` |
| **R-脱敏-7** | **替换防二次操作** — 一次 pass 内不重复替换已替换的部分（regex 用 `re.sub` 单次扫即可，天然防）|
| **R-PA-PB-V1** | **新立约归档** — Phase B UI 视觉延续性（守护者 2026-05-14 提议，悬空至今）|
| **R-LP-v3-EX-1** | **新立约归档** — Stage 2 跳过例外条款 + Codex 3 条护栏 |
