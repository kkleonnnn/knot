## 1. 三个 Bug 根因（Stage 1 已锁定）

### Bug 1 — 「📌 收藏报表」跳到管理员侧栏

**根因**：`frontend/src/Shell.jsx:39` `{active === 'chat' ? sidebarContent : adminHardcoded}` 二分。
`SavedReportsScreen` 传 `active="saved-reports"` → fallthrough 到 admin 分支，sidebarContent prop 被忽略。

### Bug 2 — 历史消息看不到 ⭐ 收藏按钮

**根因**：API 边界字段命名不一致：
- SSE `final` 事件 emit `sql` ([query.py:336](bi_agent/api/query.py:336))
- REST `/api/conversations/{id}/messages` 返回 SQLite 列名 `sql_text`
- 前端 `ResultBlock` 解构 `const { sql, ... } = msg;` → 历史消息 `sql === undefined` → `canPin=false`

**真因不是 intent NULL**（用户预判错向）：`canPin = !!(sql && Number.isInteger(msg.id) && !msg.is_saved_report)` 根本不看 intent。

### Bug 3 — 多表查询产生笛卡尔积

**三层全空**：
1. `bi_agent/services/knot/sql_planner.py::_AGENT_SYSTEM_TEMPLATE` 6 条规则零 JOIN 约束（**主路径**）
2. `bi_agent/services/llm_client.py::build_system_prompt` 同样无 JOIN 约束（备路径）
3. `bi_agent/services/knot/ohx_catalog.example.py` `TABLES` 字段只有 `db/table/topics/summary`，**无 PK/FK/relations**

**eval 盲区**：80 条 case 中 `JOIN` 0 出现；多表 case 仅 2 条且都是 simple semantics → eval-live workflow 对笛卡尔积零拦截力。

## 2. 修复方案（4 commit）

### Commit 1：Bug 2 — 后端 sql alias（资深 Stage 2 裁决）

**位置**：`bi_agent/api/conversations.py::get_messages`
**改动**：返回前对每条 message 加 `d["sql"] = d["sql_text"]`
**理由**：在 API 边界做 normalization；repo 保留 SQLite 列名干净（**职责分离**）。所有 consumer（前端 / 任何未来 client）自动拿到 `sql` 字段，与 SSE final 事件一致。

**测试增量**：`tests/integration/test_api_smoke.py` +1：拉历史消息断言 `body[0]["sql"] == body[0]["sql_text"]`。

**3b. catalog_loader 暴露 helper**（R-S3 + R-S4）：

```python
def get_relations() -> list:
    return list(getattr(_module(), "RELATIONS", []))   # R-S3 老 catalog 不抛 AttributeError

def get_relations_for_tables(selected: list[str]) -> str:
    """R-S4 按需渲染：仅返回 selected 表涉及的关联，避免 token 预算挤压。
    返回 markdown 字符串，注入 prompt。无关联时返回空字符串。"""
    rels = get_relations()
    sel = set(selected)
    matched = [r for r in rels if r[0] in sel and r[2] in sel]
    if not matched:
        return ""
    lines = ["## 表关系 RELATIONS（多表查询必须按此 ON 条件 JOIN）"]
    for left_t, left_c, right_t, right_c, sem in matched:
        lines.append(f"- `{left_t}.{left_c}` = `{right_t}.{right_c}` — {sem}")
    return "\n".join(lines)
```

```python
def _check_no_cartesian_join(case_id: str, sql: str, must_tables: list[str]):
    """v0.4.1.1: 涉及 ≥ 2 个 must_tables 的 SQL 必须有 JOIN + ON，且禁旧式 `FROM a, b`。"""
    sql_lower = sql.lower()
    matched_tables = [t for t in must_tables if t.lower() in sql_lower]
    if len(matched_tables) >= 2:
        assert re.search(r"\bjoin\b", sql_lower), f"[{case_id}] 多表 SQL 缺 JOIN: {sql}"
        assert re.search(r"\bon\b", sql_lower), f"[{case_id}] 多表 SQL 缺 ON: {sql}"
        # 检测 FROM a, b 旧式逗号 join（潜在笛卡尔积高危句式）
        assert not re.search(r"\bfrom\s+\w+(?:\.\w+)?\s*,\s*\w+", sql_lower), (
            f"[{case_id}] 检测到 `FROM a, b` 旧式 join 句式（潜在笛卡尔积）: {sql}"
        )
```
