## 1. 根因（grounded · Explore + 直查）

## 2. 决议（语义路径 metric 列中文 + metric_cols 图表硬化）

- **#5 column_labels（metric 列 → metric.display）**：`run_semantic_compile_step` 算 `column_labels = {n: _by_name[n].display or n for n in lf.metrics}`（metric.name→中文显示名）；经 **AgentResult additive 字段**带到 `query.py:452` payload；前端 `TableContainer`/`MetricCard` 表头 `{column_labels[c] || c}`（缺则 fallback 列名，LLM 路径自然 fallback）。**修最丑的 `user_position_pnl`→`持仓盈亏`**。（守护者 note：keyed lf.metrics → 仅 metric 列中文；窗口 as_name / 派生 / 维度 列表头 fallback raw 英文 = **无回归同现状**；窗口列中文 = D1-adjacent follow-on，window spec 无 display 字段）
- **numericCols 硬化（dimension_cols = lf.dimensions · 🔴 守护者 Stage 3 承重修订）**：同 AgentResult 带 `dimension_cols = list(lf.dimensions)`；前端图表 **labelCols = dimension_cols（x 轴）+ valueCols = cols − dimension_cols（全部度量：metrics + window + derived）** → 替代 `typeof number` 判断。⚠️ **不能用 lf.metrics**：窗口列输出键 = `OVER(...) AS {as_name}`（compiler.py:136）**∉ lf.metrics**、派生列同理 → 「只画 lf.metrics」会排除窗口/移动平均/派生度量不画（**回归**）；语义输出 = 维度 ∪ 度量无杂列 → `cols − dimensions` **恰 = 全部度量**。修 #190（user_id ∈ lf.dimensions → labelCol → 不画）+ **窗口/派生度量保留画（无回归）**+ 未来新度量类型自动 valueCol。`dimension_cols` 空（LLM 路径 / 非命中）→ fallback 现有 `isNumericCol` typeof 行为（byte-equal）。
- **承载**：AgentResult **+2 additive 字段** `column_labels: dict={}` + `dimension_cols: list=[]`（默认空；语义步骤填，LLM/sync 步骤不填 → byte-equal）。
- **守护者 flag 2 严守**：compiler.py SQL alias **0 改**（metric.name 仍作别名，是 HAVING/window/qualify 引用锚）；column_labels 是**纯 payload 显示层 map**，不入 SQL。
