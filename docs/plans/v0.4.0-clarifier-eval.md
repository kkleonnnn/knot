## 2. 7 类 Intent 定义与 Prompt 注入规范

### 2.1 Intent 枚举（与 eval cases.example.yaml `expects.intent` 字段一致）

| Intent | 中文 | 触发场景 | 典型 SQL 形态 | 前端 Layout |
|---|---|---|---|---|
| `metric` | 单一指标 | "昨天 GMV"、"本月新增用户数" | 1 行 1 列聚合值 | 大数字卡片 + 解读 |
| `trend` | 时间趋势 | "最近 7 天日活"、"本月每日 GMV" | 多行 + 时间维度 + 1-3 个数字列 | line chart 时间轴升序 |
| `compare` | 对比 | "昨天 vs 上周同期 GMV"、"渠道 A vs B 转化率" | 固定 N 行（每行一个对比对象）| bar chart 双系列或多系列 |
| `rank` | 排序 Top/Bottom N | "昨天充值 Top 10 用户"、"最低留存渠道 Top 5" | ≤ 50 行 + 排序明确 | 横向 bar + 表格双视图 |
| `distribution` | 分布 / 占比 | "用户按消费档次分布"、"各渠道用户占比" | 行数 = 桶数（≤ 20）| pie chart / histogram |
| `retention` | 留存矩阵 | "周留存"、"次日 / 7 日 / 30 日留存" | 矩阵形态（行×列）| heatmap-style 表格 |
| `detail` | 明细列表 | "列出昨天注册用户 ID"、"最近 10 条订单"、"列出 active=1 的数据源" | 多行多列原始数据 | 纯表格 + **CSV 导出按钮** |

### 2.2 Clarifier Prompt 注入规范

修改 `bi_agent/services/knot/orchestrator.py::_CLARIFIER_SYS`，在原 JSON 输出 schema 后**新增**：

```
新增字段 intent（必填）：

意图分类（7 类，必选其一）：
- metric         : 用户问"是多少 / 总和 / 平均"，期望单一聚合值
- trend          : 用户问"近 N 天/最近一段时间 / 每日/每周/每月趋势"，期望时间序列
- compare        : 用户问"A vs B / 同比 / 环比 / 对比"，期望两组或多组对照
- rank           : 用户问"Top N / 最大/最小/前几 / 排行"，期望排序后取前 N
- distribution   : 用户问"分布 / 占比 / 各 X 多少"（不强调排序），期望桶式聚合
- retention      : 用户问"留存 / 次日/7日/30日活跃"，期望留存矩阵
- detail         : 用户问"列出 / 列表 / 明细 / ID / 给我 X 条"，期望原始记录

判定优先级：detail > retention > rank > compare > trend > distribution > metric
（多种语义共存时按优先级取 leftmost match）

特殊规则：
- 用户提及"导出 / 下载 / CSV / 表格" → 强制 detail
- 用户提"上述/这些用户/这些 ID"等代词指代具体记录 → 通常 detail
- 单一聚合 + 按时间分桶（如"每天的 GMV"）→ trend，不是 metric

输出 JSON 结构升级：
{
  "is_clear": true/false,
  "clarification_question": null | "...",
  "refined_question": "...",
  "analysis_approach": "...",
  "intent": "metric"  // ← 新增；7 类之一，必填
}
```

### 2.3 Few-shot 示例（注入 Clarifier prompt 提升分类准确率）

每类 intent 1 条 few-shot，附在 prompt 末尾：

```
示例对照表：
Q: "昨天的 GMV 是多少？"           → intent: metric
Q: "最近 7 天每日 GMV"            → intent: trend
Q: "本周 vs 上周 GMV 对比"         → intent: compare
Q: "昨天充值金额 Top 10 用户"      → intent: rank

Q: "上周注册用户的 7 日留存"       → intent: retention
Q: "列出昨天注册的用户 ID"         → intent: detail
```

### 2.4 容错降级

- Clarifier 返回 JSON 缺 `intent` 字段 / 不在 7 类内 → 默认 `detail`（保守选择，至少不强行画图）
- 历史消息回放（v0.4.0 之前的 conversations）`intent` 为 null → 前端走现有启发式（兜底）

## 4. CSV 导出 Service 接口设计

### 4.1 Stream vs File 决策

**采用：内存 BytesIO 模式（介于纯 stream 和落盘 file 之间）**

| 模式 | 内存占用 | Excel 用户体验 | 实现复杂度 | 升级到 xlsx 难度 |
|---|---|---|---|---|
| 纯 stream（generator） | 极低 | 中（需等完整下载）| 中 | **高**（openpyxl 不支持 streaming） |
| 落盘临时文件 | 低 | 高 | 中（需清理） | 低 |
| **内存 BytesIO** ✅ | 中（≤ 1MB / 单查询）| 高 | 低 | 低 |

理由：
- `MAX_RESULT_ROWS=500` 锁死了单次结果集上限，CSV 文件 ≤ 200KB
- BytesIO 既不落盘也不流式，FastAPI `StreamingResponse(BytesIO)` 一行包装
- v0.4.1 升级到 xlsx 时直接换 openpyxl 写入同一 BytesIO，调用方不变

### 4.2 Service 层接口

新建 `bi_agent/services/export_service.py`：

```python
"""export_service — v0.4.0 detail intent 触发的 CSV 导出。
v0.4.1 计划扩展 xlsx / 富文本 / PNG。"""
from io import BytesIO, StringIO
import csv

def rows_to_csv_bytes(rows: list[dict], cols: list[str] | None = None) -> bytes:
    """把 [{col: val}, ...] 转成 CSV 字节流。
    - cols=None：从首行 keys 推断
    - 含中文 → utf-8-sig（带 BOM，Excel 直开）
    - 复杂值（dict/list）JSON 序列化
    """
    if not rows:
        return b""
    cols = cols or list(rows[0].keys())
    sio = StringIO()
    writer = csv.DictWriter(sio, fieldnames=cols, extrasaction="ignore")
    writer.writeheader()
    for r in rows:
        writer.writerow({c: _stringify(r.get(c)) for c in cols})
    return sio.getvalue().encode("utf-8-sig")


def _stringify(v):
    if v is None: return ""
    if isinstance(v, (dict, list)):
        import json
        return json.dumps(v, ensure_ascii=False, default=str)
    return str(v)
```

### 4.3 API 接口

新增路由 `bi_agent/api/exports.py`：

```python
"""导出路由（v0.4.0 仅 CSV；v0.4.1 加 xlsx）。"""
from io import BytesIO
from fastapi import APIRouter, Depends, HTTPException
from fastapi.responses import StreamingResponse
from urllib.parse import quote

from bi_agent.api.deps import get_current_user
from bi_agent.repositories.message_repo import get_messages
from bi_agent.repositories.conversation_repo import list_conversations
from bi_agent.services.export_service import rows_to_csv_bytes

router = APIRouter()


@router.get("/api/messages/{message_id}/export.csv")
async def export_message_csv(message_id: int, user=Depends(get_current_user)):
    """导出某条 message 的 rows 为 CSV。
    权限校验：必须是该 conversation 的所有者（admin 例外）。"""
    # 1. 查 message + 校验权限
    # 2. CSV 编码 → BytesIO
    # 3. StreamingResponse + Content-Disposition
    msg = _find_message_for_user(message_id, user)
    if not msg:
        raise HTTPException(404, "消息不存在或无权访问")
    if not msg.get("rows"):
        raise HTTPException(400, "该消息无可导出数据")

    csv_bytes = rows_to_csv_bytes(msg["rows"])
    filename = f"export_msg{message_id}.csv"
    headers = {
        "Content-Disposition": f"attachment; filename*=UTF-8''{quote(filename)}",
    }
    return StreamingResponse(BytesIO(csv_bytes), media_type="text/csv; charset=utf-8", headers=headers)
```

## 9. 风险与回滚

| 风险 | 触发条件 | 缓解 | 回滚方式 |
|---|---|---|---|
| Clarifier intent 准确率 < 90% | live eval 红灯 | Few-shot 示例迭代 + prompt 微调；不达标延期到 v0.4.0.1 | 不合 PR；继续迭代 prompt |
| 前端组件回归（旧消息渲染破坏）| 手工验证发现 | 启发式降级路径 `inferIntentFromShape` 必须保留兼容 | revert frontend 改动 |
| CSV 中文乱码 | utf-8 vs utf-8-sig | 统一 utf-8-sig；test_export_service 验证 BOM | revert export_service |
| 导出权限漏洞 | analyst 拿到他人 message_id | api/exports.py 必须做 conversation 所有者校验（含 admin 例外） | 紧急 hotfix 改为强校验 |
| live CI secret 配置错误 | eval-live 始终红 | 先 workflow_dispatch 手动跑一次验证 | secret 修正 |
