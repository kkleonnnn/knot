## 三、给 SRE 的提醒（影响 KNOT 部署成败的环境因素）

### ① futures-admin 网络可达性
KNOT 容器需要能 HTTP GET `<futures-admin host>`。请确认：
- futures-admin 是否在同 k8s cluster？
  - 同 cluster：推荐用内部 service DNS（更安全 + 低延迟）
  - 不同 cluster / 外部：需要 NetworkPolicy egress 允许该域名
- 推荐用内部 DNS 是因为：不暴露公网域名 + 不需要在公网白名单加 IP

### ② 日志已自动 JSON 格式（Filebeat / Kibana 友好）
KNOT v0.6.1.11 起，pod 跑在 k8s 里（non-tty 环境）日志自动是 JSON line，顶层字段：
```
time / level / msg / logger / func / line / request_id
```
异常时多 `exception` 子对象。

**提醒**：如果 Filebeat 当前是按文本格式抓的，需要让 Ann 加 `decode_json_fields` 处理器把 message 字段解到顶层。

### ③ DB schema 自动迁移
启动期自动 `ALTER TABLE data_sources ADD COLUMN http_config`（幂等）。无需手动 SQL。
不影响现有数据。

### ④ 回滚成本
零停机方案完整：image tag 回旧版即可，DB 新加列对旧版无影响，env / ConfigMap 多出来的也不影响旧版。
