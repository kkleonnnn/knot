## 7. PATCH 节奏（参考 v0.4.5 / v0.4.6 经验）

每 PATCH ≈ 5 commits / 1 周内完成：
- Day 1：plan LOCKED commit
- Day 2-4：commits #1-#5（每 commit TDD + 闸门 + 守护者复核）
- Day 5：amend 修补 + PR + tag + release

**不要急**。守护者反复强调 v0.4.x 教训：
- TDD 不可省（先红再绿）
- 每 commit 三闸门必须全绿（pytest + lint-imports + ruff）
- frontend build 改前端时必跑
- CI 必绿才合（v0.4.5 曾因 CI 没注入 master key 红过一次）
- 完成 commit 立即停下汇报，不自动续推
