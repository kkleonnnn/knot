# v0.7.24 — clarifier 数字选项断片修复（Part A：澄清选项进 history · #2 · Stage 1 草案）

> **触发**：价值自测 #2。clarifier 弹编号选项澄清（「①当前持仓 ②本月已平仓 ③本月成交」）→ 用户回纯数字「2」

## 1. 根因（grounded · Explore）

4 环节，bug 在 ②③（数据流丢选项，非 prompt）：
- ① 澄清问题**落库** ✅ `query.py:209-210`：is_clear=false 分支 `save_message(question=req.question, sql="", explanation=cq, agent_kind="clarifier")` — 选项文本存 `explanation` 字段。
- ② history 构建**丢 explanation** 🔴 `query.py:74-75`：`history = [{"question", "sql": sql_text, "rows"} ...]` — **explanation 被忽略**。
- ③ clarifier 渲染**再过滤** 🔴 `clarifier.py:51-56` `_render_clarifier_inputs`：sql 空时只 `lines.append(f"Q: {q}")` — 选项彻底丢 → clarifier prompt 的 `{history}` 里没上轮选项。
- ④ prompt **有** L25「已有澄清回复融入 refined_question」但**无数字选项专项**（clarifier.md:25 + 代词解析 :56-61）。本地+线上 clarifier prompt **均含 L25**（Explore 备份 `clarifier_db_custom_backup.md` L55 确认 2026-05-25 定制版也有）。

→ 用户回「2」时 clarifier `{history}` 只有 `Q: 本月各交易对持仓盈亏`（无选项）+ 当前问题「2」→ 无法连接 → is_clear 误判 → 断片。

## 2. 决议（Part A only · 纯代码数据流 · 0 clarifier.md）

- **A1（query.py:74-75 history dict）**：+`"explanation": m.get("explanation", "")` +`"agent_kind": m.get("agent_kind", "")`（additive；其它消费者忽略额外字段）。
- **A2（clarifier.py:51-56 `_render_clarifier_inputs`）**：sql 空分支，若 `agent_kind == "clarifier"` 且 explanation 非空 → 渲染上轮澄清选项（`Q: {q}\n  （上轮澄清选项：{explanation}）`）；**非 clarifier 的 sql 空轮（如错误轮）+ sql 非空轮渲染 byte-equal**（不动）。
- **靠既有 L25 解析**：选项进 history 后，clarifier prompt 既有 L25「如对话历史中已有澄清回复，直接视为 is_clear=true 并将澄清信息融入 refined_question」→ 期望自行把「2」+ 可见选项 → 解析成选项②，融入 refined_question。**0 clarifier.md 改 → 0 re-seed**（本地+线上既有 L25 都在）。
- **Part B（clarifier.md 数字选项专项指令）conditional defer**：仅当 live 测「2」仍不解析（L25 不够）→ 才加（会引入 re-seed gotcha）。本刀不做。
