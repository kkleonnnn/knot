## §0 背景 — 白屏验收实测暴露的串联根因 + R-2FA 不变量

强制 2FA（v0.6.5.0/.1，deps.py:140-147 默认 on）落地后，「账号密码登录 → 强制绑定 TOTP」在真实使用暴露 **8 个 bug**（6 白屏链 + 2 活体契约），互相串联放大成「整屏白屏 + enroll 卡死 + 已 enrolled 登不进 + 救援通道断」：

- **F1（白屏直接元凶）** `knot/api/_rate_limit.py` 共 **4 处** `raise HTTPException(429, detail={"ja":..,"zh":..})`（L94-100 / L127-134 / L143-150 / L163-170）。全站唯一 `{ja,zh}` 结构（`grep '"ja"'` 仅命中这 4 行）。
- **F3（白屏机理）** `frontend/src/api.js:36/38`：`err.message` 做了 `JSON.stringify` 兜底，但 **`err.detail` 保留原始对象**（L38 + L68 verify 分支）。`Enroll.jsx:38/53` `setError(err.detail||..)` 把 `{ja,zh}` 塞进 state → `ErrorBanner`(Enroll.jsx:222 `{msg}`) 渲染对象 → React #31「Objects are not valid as a React child」→ 整屏白屏（全仓无 ErrorBoundary）。
- **F2（enroll 卡死）** `totp.py:77`(init) 与 `:103`(complete) 共用 `enforce_totp_enroll_rate_limit`，同 key `totp_enroll:{uid}`，同 3/3600 桶。一次正常绑定耗 2/3；屏 remount + 错码重试快速触顶 → complete 429 → 卡死 1 小时。
- **F5（扫码全失败）** `totp_service` 每次 enroll_init 生成全新 secret 不持久化；`Enroll.jsx:29-42` mount(deps `[]`) 每次重调 init → 刷新换 secret → 认证器旧条目动态码与最新 secret 全错位 → enroll-complete 必败（实测扫 3+ 码全失败）。
- **F6（发版白屏）** `main.py:230` SPA catch-all 裸 `FileResponse(index.html)` 无 Cache-Control → 浏览器启发式缓存旧 index.html → 引用已删旧 hash bundle → `/assets/index-<旧hash>.js` 404 → 白屏。
- **⭐ C1（verify P0 · 契约审计坐实）** `api.js:59-63` verify 把 `interim_token` 放 **HTTP header**（`_hWith`），body 仅 `{code}`；但 `schemas.py:27-30 TotpVerifyRequest` 要求 `interim_token` 在 **body 且必填**，`totp.py:116/122` 从 `req.interim_token` 读、**verify 端点无 `Depends(get_current_user)` 不读 header**。⟹ 所有已 enrolled 用户登录第二步 **必 422**（Pydantic 校验层即报 field required）→ 拿不到完整 JWT → **登录系统整体瘫痪**（你现在只能靠 `KNOT_TOTP_BYPASS_ADMIN` 后门进就是因为它）。
- **⭐ C2（reset P1 · 契约审计坐实）** `api.js:73` reset 发 `{user_id}`，但 `schemas.py:33-35 TotpResetRequest` 要求 `target_user_id`（必填）、`totp.py:165` 读 `req.target_user_id`。⟹ admin 点「重置 TOTP」**必 422** → 无法为丢失手机/被锁用户救援 → 结合 C1，2FA 故障无管理员侧自救手段。
- **F4（R-2FA 不变量未落地 + 退出卡死 + FOUC）** —— 见下方 R-2FA 专段（原设计「mount me() 清 token」经审计证实为**死代码**，本批彻底重设为**后端 rollout bump**）。

**契约审计完整性结论**（独立 4-agent workflow · grounded 全 17 routers + 全 body 端点）：**除 C1/C2 外 0 个第三同类契约 bug**；另 2 个 P2 非阻断（见 §2 P2 段）。

### R-2FA 不变量（资深 2026-06-19 directive · verbatim）
> 「默认运维更新后，之前已经登录了的用户全部需要重新登录，登录后再让他们绑定，总之不可以再出现先扫码再输账户密码再扫码的情况。」

**审计证实的真实行为模型**（A3 agent · grounded deps.py:94/144 + auth.py:30-49 + App.jsx:42/62）：
- `me()`(`/api/auth/me`) 命中 enroll-gate 白名单前缀 `/api/auth/`（deps.py:144 复用 L94 常量）⟹ **未 enroll 用户 me() 返 200，永不返 403 totp_enroll_required**。
- 故原设计 F4「mount me() catch 403 → 清 token」是**死代码**（分支恒不执行）。
- 真正 enroll-redirect 发生在 user-ready 后的 **prefetch onErr**（App.jsx:62；`/api/conversations` 等非白名单端点 403）。
- ⟹「旧 pre-2FA session 刷新即跳过登录直接进 Enroll」无法靠前端 mount 逻辑修；**正确机制 = 后端在 rollout 时一次性失效所有旧 JWT**（资深拍板 option A）。

**机制**（复用既有 token_version 吊销链 deps.py:121-127）：
