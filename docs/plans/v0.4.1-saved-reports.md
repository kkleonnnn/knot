## 2. 数据库 Schema 设计（含 R-S2）

### 2.1 新表

`bi_agent/repositories/schema.sql` 追加：

```sql
CREATE TABLE IF NOT EXISTS saved_reports (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id             INTEGER NOT NULL,
    source_message_id   INTEGER,                                  -- 仅溯源；不做硬外键（R-S7）
    data_source_id      INTEGER,                                  -- R-S2 重跑必用；NULL = fallback 当前默认 + warning
    title               TEXT    NOT NULL,
    question            TEXT,                                     -- 原始 NL 问题快照
    sql_text            TEXT    NOT NULL,                         -- 冻结 SQL，重跑入口
    intent              TEXT,                                     -- v0.4.0 7 类之一；快照（R-S6 兜底 'detail'）
    display_hint        TEXT,                                     -- INTENT_TO_HINT 快照（R-S4 优先级 #1）
    pin_note            TEXT,
    last_run_at         TEXT,
    last_run_rows_json  TEXT,                                     -- 软限制 200 行（R-3 防御性）
    last_run_truncated  INTEGER DEFAULT 0,                        -- 1 = rows 超 200 行被截断
    last_run_ms         INTEGER DEFAULT 0,
    pinned_at           TEXT    DEFAULT (datetime('now','localtime')),
    UNIQUE (user_id, source_message_id)                           -- R-12 幂等键
);
```

### 2.2 关键设计点

- **不加硬 FOREIGN KEY**（资深 Redline #2）：source_message_id / data_source_id 仅逻辑关联；上游删除时 saved_report 仍存活（已 snapshot 全部必要字段）
- **UNIQUE (user_id, source_message_id)** 是 R-12 幂等键：重复 pin → service 层 INSERT OR IGNORE + 回查既存行返回
- **data_source_id 可为 NULL**：手工建报表场景或 source 删除后；执行重跑时 fallback `engine_cache.get_user_engine(user)` + 前端 banner 警告
- **last_run_rows_json 软限制**：`saved_report_service.run` 内 `rows = rows[:200]`；超出设 `last_run_truncated=1`；前端在详情页显示"仅展示前 200 行预览"

### 2.3 迁移策略

`base.py::init_db` 不需要 ALTER（CREATE TABLE IF NOT EXISTS 兜底）。schema.sql 单点维护。

## 3. snapshot 流程（pin from message）

### 3.1 调用链

```
POST /api/messages/{message_id}/pin
    body: { title?: str, pin_note?: str }
  ↓
api/saved_reports.pin_from_message:
  1. msg = message_repo.get_message(message_id)            ← v0.4.0 已有
  2. owner_id = conversation_repo.get_conversation_owner(msg["conversation_id"])
     权限：owner_id == user["id"] OR user.role == "admin"  否则 404 防枚举
  3. data_source_id = _resolve_pin_data_source(user, msg)  ← 见 §3.3
  4. saved_report_service.create_from_message(...) → 返回 saved_report dict
  5. 回包 200 OK（含 already_pinned: bool 标记给前端）
```

### 3.2 service 层 snapshot 逻辑

`bi_agent/services/saved_report_service.py::create_from_message`：

```python
def create_from_message(
    user_id: int, msg: dict,
    title: str | None, pin_note: str | None,
    data_source_id: int | None,
) -> tuple[dict, bool]:
    """返回 (saved_report, already_existed). 幂等：UNIQUE 冲突回查既存。"""
    intent = msg.get("intent") or "detail"                       # R-S6
    hint = INTENT_TO_HINT.get(intent, "detail_table")
    rows = msg.get("rows") or []
    truncated = len(rows) > 200
    rows_snap = rows[:200] if truncated else rows                # R-3 软限制

    final_title = _default_title(title or msg.get("question"))   # R-S5

    existing = saved_report_repo.get_by_unique(user_id, msg["id"])
    if existing:
        return existing, True                                    # R-12 幂等

    rid = saved_report_repo.create(
        user_id=user_id, source_message_id=msg["id"],
        data_source_id=data_source_id,
        title=final_title, question=msg.get("question"),
        sql_text=msg.get("sql_text") or "",
        intent=intent, display_hint=hint, pin_note=pin_note,
        last_run_rows_json=json.dumps(rows_snap, ensure_ascii=False, default=str),
        last_run_truncated=1 if truncated else 0,
        last_run_ms=msg.get("query_time_ms") or 0,
        last_run_at=msg.get("created_at"),                       # 用 message 时间作首次 last_run
    )
    return saved_report_repo.get(rid), False


def _default_title(s: str | None) -> str:                        # R-S5
    q = (s or "").strip() or "未命名报表"
    return q[:30] + ("…" if len(q) > 30 else "")
```

### 3.3 data_source_id 解析（R-S2）

pin 时尽量记录用户当前在用的 source。**当前 message 没存 data_source_id 字段**（v0.4.0 messages 表不含），所以 v0.4.1 用以下推断：

```python
def _resolve_pin_data_source(user: dict, msg: dict) -> int | None:
    """v0.4.1 fallback 链：default_source_id → user_sources 唯一一条 → None
    （None 表示重跑时走 get_user_engine 当前默认）"""
    if user.get("default_source_id"):
        return user["default_source_id"]
    src_ids = data_source_repo.get_user_source_ids(user["id"])
    if len(src_ids) == 1:
        return src_ids[0]
    return None  # 多源场景 / 无源场景：重跑时 fallback + warning
```

> **未来增强（v0.4.2+）**：messages 表加 `data_source_id` 列，pin 时直接读，更精确。本 PATCH 不改 messages 表（避免再次 ALTER 引入回归）。

## 4. 重跑路径（冻结 SQL）

### 4.1 路由

`POST /api/saved-reports/{id}/run` → 返回 `{rows, last_run_at, last_run_ms, error?, warning?}`

### 4.2 service 流程

```python
def run(saved_report_id: int, user: dict) -> dict:
    sr = _load_owned(saved_report_id, user)                  # 权限校验
    engine, warning = _resolve_run_engine(sr, user)          # R-S2
    t0 = time.time()
    try:
        rows, db_error = db_connector.execute_query(engine, sr["sql_text"])
    except Exception as e:
        rows, db_error = [], str(e)[:200]
    elapsed_ms = int((time.time() - t0) * 1000)

    truncated = len(rows) > 200
    rows_snap = rows[:200] if truncated else rows
    saved_report_repo.update_last_run(
        saved_report_id,
        rows_json=json.dumps(rows_snap, ensure_ascii=False, default=str),
        truncated=1 if truncated else 0,
        elapsed_ms=elapsed_ms,
    )
    return {
        "rows": rows_snap, "truncated": truncated,
        "last_run_ms": elapsed_ms, "last_run_at": _now_iso(),
        "error": db_error, "warning": warning,
    }


def _resolve_run_engine(sr: dict, user: dict):
    """R-S2: data_source_id 优先；若 source 已删 / 用户失去关联 → fallback + warning"""
    sid = sr.get("data_source_id")
    if sid:
        src = data_source_repo.get_datasource(sid)
        owned = sid in data_source_repo.get_user_source_ids(user["id"])
        if src and src["is_active"] and (owned or user.get("role") == "admin"):
            engine = engine_cache.get_engine_for_source(sid)
            if engine is not None:
                return engine, None
        # fallback
        engine, _ = engine_cache.get_user_engine(user)
        return engine, "原数据源已不可用，已切换到当前默认数据源（口径可能不一致）"

    engine, _ = engine_cache.get_user_engine(user)
    return engine, None
```

> **engine_cache 需新增** `get_engine_for_source(source_id) -> engine | None`：按单个 source 取/建 engine（不依赖 user → user_sources 链）。如果 engine_cache 已有等价函数（命名可能不同）直接复用。**执行手册第一步必须 grep 确认。**

@router.get("/api/saved-reports/{report_id}/export.csv")
async def export_saved_report_csv(report_id: int, user=Depends(get_current_user)):
    sr = saved_report_service.get_owned(report_id, user)     # 404 防枚举

### 6.4 R-S4 — `effectiveHint` 三级优先级链

`Chat.jsx` 顶部新加 helper：
```js
function resolveEffectiveHint(msg, rows, cols) {
  if (msg.display_hint) return msg.display_hint;             // 1. saved_report 快照（v0.4.1）
  if (msg.intent && INTENT_TO_HINT[msg.intent]) {            // 2. v0.4.0 message + 当前 mapping
    return INTENT_TO_HINT[msg.intent];
  }
  return INTENT_TO_HINT[inferIntentFromShape(rows, cols)]    // 3. 启发式（v0.4.0 之前老消息）
      || 'detail_table';
}
```

`ResultBlock` 内当前 `const layoutHint = INTENT_TO_HINT[effectiveIntent] || 'detail_table'` 改成走 `resolveEffectiveHint(msg, rows, cols)`。

## 10. 风险与回滚

| 风险 | 触发条件 | 缓解 | 回滚 |
|---|---|---|---|
| R-S2 多源 fallback 数据不一致 | 用户 pin 时 source A，重跑时 fallback 到 source B | warning banner + 强烈提示用户检查口径 | revert _resolve_run_engine 改为强校验 fail |
| 重跑触发 SQL guardrail 拒绝（schema 变了 / 字段消失） | 业务库演进 | service 捕获 db_error 进 last_run_*；前端展示错误 banner | 无须 revert；用户重新发起对话即可 |
