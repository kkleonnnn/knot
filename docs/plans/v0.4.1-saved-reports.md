## 2. saved_reports schema 设计（资深 R-3 决议）

### 2.1 表结构

```sql
CREATE TABLE IF NOT EXISTS saved_reports (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id         INTEGER NOT NULL,
    name            TEXT    NOT NULL,
    payload         TEXT    NOT NULL DEFAULT '{}',  -- JSON; SQLite 用 TEXT 存
    tags_json       TEXT    DEFAULT '[]',           -- 冗余拆出便于过滤检索
    created_at      TEXT    DEFAULT (datetime('now','localtime')),
    updated_at      TEXT    DEFAULT (datetime('now','localtime')),
    last_run_at     TEXT
);

CREATE INDEX IF NOT EXISTS idx_saved_reports_user ON saved_reports(user_id);
```

> **资深 R-3 决议**：表 + JSONB 字段，BI 结果形态多变（Metric/Table/ECharts Config），
> 用 `payload` 单字段存储 + `tags_json` / `name` / `last_run_at` 拆出做检索维度，
> 避免频繁 schema migration。

### 2.2 payload schema 版本管理

```python
# bi_agent/models/report.py
@dataclass
class SavedReportPayload:
    schema_version: int = 1   # ⭐ 关键：未来加字段时 +1，repo 层做 migrate
    question: str = ""
    refined_question: str = ""
    intent: str = "detail"
    display_hint: str = "detail_table"
    sql: str = ""
    rows_snapshot: list = field(default_factory=list)
    last_executed_at: Optional[str] = None
    model_key: str = ""
    data_source_id: Optional[int] = None
    history_context: list = field(default_factory=list)
    tags: list = field(default_factory=list)
```

**未来兼容策略**：repo 层读取时检查 `schema_version`，若旧版（如 1 → 2 时新增字段）做 in-memory 升级；写入永远用最新版。

### 2.3 与 v0.4.0 的对接点

`payload.intent` + `payload.display_hint` 必须**与 v0.4.0 输出的 message 一致**。
保存路径：

```python
# api/reports.py::POST /api/reports
def save_report(message_id: int, name: str, user):
    msg = message_repo.get_by_id(message_id)
    assert msg["conversation_id"] in user_conv_ids   # 权限校验
    payload = {
        "schema_version": 1,
        "question": msg["question"],
        "intent": msg["intent"],                          # ← v0.4.0 已存
        "display_hint": INTENT_TO_HINT[msg["intent"]],    # ← 重新映射
        "sql": msg["sql_text"],
        "rows_snapshot": msg["rows"],
        ...
    }
    return report_repo.create(user_id=user["id"], name=name, payload=payload)
```

## 3. "重跑"流程设计（资深 R-7 红线）

### 3.1 重跑必须经过 3-Agent 完整流程

```
POST /api/reports/{id}/run
  ↓
load report → 取 payload.question + payload.intent + payload.history_context
  ↓
[关键] 走 services/knot/orchestrator.run_pipeline()
  → Clarifier (复用原 history_context)
  → SQL Planner (复用原 data_source_id)
  → Presenter
  ↓
更新 payload.rows_snapshot / sql / last_executed_at
报表 last_run_at = now
返回新结果
```

### 3.2 为什么不走捷径

错误做法（**禁止**）：
```python
# ❌ 不可：直接拿 payload.sql 重跑
rows = db_connector.execute_query(engine, report.payload.sql)
```

理由：
- 业务库 schema 可能变（字段重命名 / 表迁移）
- intent 决策可能因为时间口径变化而需要重新做（"昨天" 的含义每天都在变）
- 用户当时的 history_context 可能让查询有上下文依赖
- 走完整流程才能保证"所见即所得"的真实复现

### 3.3 失败降级

重跑失败时（业务库连接失败 / SQL 字段不存在）：
- 不修改 payload.rows_snapshot（保留上次成功结果）
- 返回 HTTP 200 + `{"success": false, "error": "...", "stale_snapshot": true}`
- 前端显示警告 banner "重跑失败，显示上次结果（YYYY-MM-DD）"

## 4. 完整导出（v0.4.0 CSV → v0.4.1 三种）

### 4.1 xlsx（后端实现）

扩展 `bi_agent/services/export_service.py`：

```python
def rows_to_xlsx_bytes(rows: list[dict], cols: list[str] | None = None,
                       sheet_name: str = "Data") -> bytes:
    """返回 xlsx 字节流；单 sheet（v0.4.x 范围）。"""
    from io import BytesIO
    from openpyxl import Workbook
    wb = Workbook()
    ws = wb.active
    ws.title = sheet_name[:31]   # Excel sheet name 限制 31 字符

    cols = cols or list(rows[0].keys()) if rows else []
    ws.append(cols)
    for r in rows:
        ws.append([_stringify_for_xlsx(r.get(c)) for c in cols])

    bio = BytesIO()
    wb.save(bio)
    return bio.getvalue()
```

## 8. 风险与回滚

| 风险 | 触发条件 | 缓解 | 回滚 |
|---|---|---|---|
| payload schema 字段遗漏 | 报表打开渲染失败 | schema_version=1 + dataclass 默认值兜底；前端 render 容错 | 修 dataclass 默认值；不需要 DB migrate |
| 重跑时业务库 schema 变化 | SQL planner 失败 | §3.3 失败降级显示 stale_snapshot | 不动 payload；提示用户改报表口径 |
| openpyxl 依赖膨胀 | requirements 增加 | 已在 v0.2.x 引入；本 PATCH 不新增依赖 | n/a |
| xlsx 中文乱码 | utf-8 处理不当 | openpyxl 默认 utf-8；test 必须断言中文字段 | 修 _stringify_for_xlsx |
| ResultRender 抽离破坏 v0.4.0 layout | Chat 页旧消息渲染破 | 抽离前先在 storybook 风格 mock 各 intent 验证；保留 ResultBlock 兼容入口 | revert frontend |
| saved_reports 索引不当 | 列表查询慢 | 已加 idx_saved_reports_user；后续按 tags 查再加 | 加索引（增量） |
| 重跑触发的 LLM cost 失控 | 用户暴力重跑 | 加用户级"24 小时内同一报表重跑 ≤ 10 次"软限制（响应头 warning，不阻断） | 调高 / 调低阈值 |
