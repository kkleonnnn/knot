## §2 决策点（资深拍板项 · 执行者带推荐）

### D1 — 范围（推荐：窄）
- **✅ 推荐 = F4 re-run + F2 徽标**（闭合 LogicForm 观测/修正故事）。
- **推迟 v0.7.5+**：LogicForm 版本历史/diff（侧表 is_corrected/parent 已为此预留，但 diff UI 是独立卖点）+ 事件/规则/动作（5 层语义上层，高风险逐层放，独立大刀）。
- 理由：延续 v0.7.0/.2/.3「更窄」哲学；re-run 是高风险面，不混入更大 scope。

### D2 — ⭐ re-run engine 解析（承重 fork · v0.7.3 盲区同构续集）
侧表无 engine 绑定 → re-run 真执行「用哪个 engine」必须拍：

| 选项 | 机制 | 利 | 弊 |
|---|---|---|---|
| **(A) 原 message 用户**（✅ 推荐）| message→conv→`user_id`→`get_user_engine(原用户)` | 自包含（无 schema 改）+ **数据保真**（同原查询所见数据源）+ 与 live 路径同 engine 解析 | admin 经原用户数据源执行（单租户下数据源共享；admin 已特权 + 审计）|
| (B) admin 自己 engine | `get_user_engine(admin)` | 隔离最简 | **数据错**（admin 数据源 ≠ 原查询；SQL 引用表可能不在 admin schema）|
| (C) 侧表存 source_id | audit 时存数据源（仿 saved_reports）| 最显式 | 需 schema 加列 + **仅惠及未来行**（存量无 source）|

**推荐 (A)**：口径保真（R-SL-40 原 catalog 重编译）+ 数据保真（原用户 engine）双全；自包含无 schema 改。

### D3 — re-run = 独立端点（推荐 ✅）
`POST /logicform-audit/{audit_id}/rerun`：从侧表行 `logicform_json` + `catalog_id` **重编译**（复用 /correct 逻辑，R-SL-40 原 catalog）→ 解析 engine（D2）→ `execute_query` → 返 rows。
- `/correct` **保持 compile-only**（资深 v0.7.3 安全线不动）；re-run 是**新的、独立 gate + 独立审计 + 独立风险复核**的执行能力。
- 对**任意审计行**统一（原行 = 看当前数据；修正行 = 看修正后数据）。compile/execute 边界**显式**（正是资深 v0.7.3 画的线）。

### D4 — re-run 临时（推荐 ✅）
返 rows 给 admin UI，**不在用户会话写新 message**（零污染 —— admin 动作不写用户数据）。修正血缘行（is_corrected）已由 /correct 落；re-run 只给 admin 看数据。

### D5 — budget/安全（推荐：复用既有 + 不加 LLM 桶）
re-run = **0 LLM**（确定性 SQL 已编译，无 parser/presenter）→ 无成本桶。DB 执行经 `execute_query` row cap + admin-only + 审计兜底。rate-limit：admin-only 端点，推荐**不加专用限流**（或轻量复用），资深可拍。

### D6 — F2 徽标数据路径（推荐 ✅）
- **live**：SSE final 已带 `engine`（v0.7.3 就绪）→ 前端 ResultBlock 读 `msg.engine` 渲染小徽标。
- **历史**：enrich `get_messages` —— 批量 LEFT JOIN/IN（Stage 2 防 N+1）；engine 派生 = **侧表有行且 `compile_error_reason` 空 = semantic；否则 llm**（守护者 Stage 3：near-miss 有侧表行但回退 LLM，**不可误标 semantic**）。
- 徽标 = **label only**（`semantic`/`llm`），**非 SQL/LogicForm**（脱敏 sustained — 非 admin 看徽标安全，看不到 SQL）。
