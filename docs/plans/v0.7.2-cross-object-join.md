## 2. 关键决策点（D1~D5 · 请资深/守护者拍板）

| # | 决策 | 推荐 | 备选 / 风险 |
|---|---|---|---|
| **D1** | **BFS 阈值（防组合爆炸）** | ⭐ **路径表数 ≤ 4 + 单条最短路径**（美团「半成品星座 + 带阈值」）；超阈 → 回退 | 无阈值 → 组合爆炸（美团踩坑）；阈值太松 → 慢/错 |
| **D2** | **fan-out 守护来源** | ⭐ **import `_is_fan_out` 经 `sql_planner` re-export**（R-100 维护的 import 面，0 重构）；compiler 复用 | (a) promote 到 sql_validator（cohesion，但 sql_validator 注释明示与 _is_fan_out「互补分离」+ 触 sql_planner_tools 重构 → 单独 chore）(b) 重实现（重复，拒） |
| **D3** | **多对象命中判定** | ⭐ **保守：唯一 BFS 路径才命中**；无路径/多路径/超阈 → None 回退。宁退化不报错 | 激进（任选一条路径）→ 路径歧义致错 SQL，拒 |
| **D4** | **caliber alias 重写**（⭐ 最硬） | ⭐ **编译器给每 base_object 分配确定性 alias（如 `t0/t1` 按对象名排序）+ 重写 caliber `o.` → `<alias>.`**（按 metric.base_object 映射；regex 或 sqlglot）；v0.7.1 单对象退化为 `o` 兼容 | (a) 改 metric caliber 约定（alias=对象名）→ 破 v0.7.0/v0.7.1 既有 metric，需迁移，拒 (b) 不重写 → 多对象 `o` 撞，错 |
| **D5** | **JOIN 类型 + fan-out** | ⭐ **INNER JOIN**（确定性，避 LEFT fan-out 语义）+ fan-out 守护检测多聚合×多 JOIN 风险 → 回退（不解决） | LEFT JOIN → fan-out 语义复杂；解决 fan-out（pre-agg）→ 难，留未来 |

## 4. 红线 R-SL-23 ~ R-SL-29（续号 · 待批准）

- **R-SL-23（BFS 路径确定性）**：`_find_join_path` 阈值内返**唯一确定路径**（最短 + 确定性 tie-break，如按表名排序）；同输入 → 同路径 → 同 SQL。守护：同对象集 BFS 2 次 byte-equal 路径。
- **R-SL-24（阈值防爆炸）**：路径表数 > N（D1，建议 4）→ raise → 回退。守护：超阈对象集 → CompileError。
- **R-SL-25（多对象命中保守）**：无路径 / 多最短路径（歧义）/ 超阈 → None 回退（R-SL-14）。守护：无路径/歧义 → 回退 LLM。
- **R-SL-26（caliber alias 重写正确）**：每 metric 的 caliber `o.` 重写为其 base_object 的分配 alias；多对象 0 alias 撞。守护：2 对象 2 metric → 各 caliber 用各自 alias byte-match。
- **R-SL-27（fan-out 守护回归）**：多表编译产出过 `_is_fan_out`；命中 → raise → 回退。守护：构造 ≥2 聚合 × ≥2 JOIN → 熔断（wiring 非空转）。
- **R-SL-28（单对象不回归）**：v0.7.1 单对象路径**行为不变**（单对象 → 无 JOIN → alias 退化 `o` → byte-equal v0.7.1 SQL）。守护：单对象 LogicForm 编译 == v0.7.1 输出。
- **R-SL-29（OOS-1 + 跨源隔离）**：BFS 只在 active catalog；跨源（http source_type）对象 → 排除/回退（不跨源 JOIN sustained）。

## 7. 验收标准

- **BFS 路径**：唯一确定（同对象集 byte-equal 路径）；超阈/无路径/歧义 → 回退（R-SL-23/24/25）。

- **OOS-1**：BFS 只在 active catalog；跨源对象排除（R-SL-29）。
