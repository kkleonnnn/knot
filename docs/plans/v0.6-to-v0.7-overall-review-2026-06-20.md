## §4 下一 MINOR 预期范围骨架（产物 4 · 非锁定）

> 详见 Stage 0 预研 `v0.7.0-semantic-layer-prestudy-2026-06-12.md`（资产衔接矩阵 + open questions 完整版）。此处仅骨架。

**定位**：v0.6.x 把 NL→SQL 直出打磨到天花板；v0.7 叠**确定性语义层** —— 已定义指标走确定性编译，未定义仍走 LLM 兜底（**混合架构**，零定义卖点不丢）。

**5 层（自下而上，可增量）**：指标 ← 对象 ← 事件 ← 规则 ← 动作。**指标层是第一刀**（质量跃迁 ROI 最高 / 风险最低）。

**关键改造**：`NL → LogicForm → SQL`（LLM 出结构，语义层确定性编译，消除 SQL 抖动 + admin 可审计修正）。

**现成地基**（v0.7 是「连地基成楼」非「平地起楼」）：`RELATIONS`→对象层 · `LEXICON`→指标映射 · `time_resolver`→事件层时间 · `budget_service`(scope/threshold/action)→规则层 · `sqlglot/sql_validator`→三层 SQL 构建 + 路径校验 · `adapters/notification+http`→动作层通道。

**增量序（非锁定）**：v0.7.0 起手 + 整体审核 + 指标注册表 → v0.7.1 LogicForm + 单对象编译（⭐ 最高 ROI）→ v0.7.2 对象层跨表 JOIN → v0.7.3 混合路由 + 审计 UI → v0.7.4+ 事件/规则/动作（高风险逐层放）。

**v0.7.0 第一刀（仲裁 B.4 更窄）**：指标注册表 + admin UI + **加密决策 + 审计接线 + 成本分桶 + async-native 六位一体**（不变量与 schema/CRUD 同址而生）；LogicForm 编译留 v0.7.1。

**硬前置**：① v0.6.x 收官（含 §3.1 admin.py 拆分 + catalog.py 单独拆 + check_file_sizes 根治）② 本仪式 4 产物 LOCKED（含 §4.5 不变量带入清单）③ 完整 v3 三阶段（业务模型重构级，**不适用简化协议**）。
