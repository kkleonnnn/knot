## §0 背景 · 当前缺陷 · 资深裁定

**当前缺陷（ground-verified）**：
- `_admin_bypass_active()` 优先级 2 =「0 admin enrolled → 自动 bypass」。但**没有自愿 enroll UI**（仅强制流程 + admin 给别人 reset）⟹ **唯一/首个 admin 永远无法被 enroll，2FA 对 admin 形同虚设**。
- `KNOT_TOTP_REQUIRED` 默认 unset（off）→ 内测期不强制（原 R-PA-8 设计「公测三方会议后才 on」）。

**资深裁定（AskUserQuestion 锁定）**：
1. admin 不豁免 → 删优先级 2 bootstrap bypass。
2. `KNOT_TOTP_REQUIRED` 默认 on（提前 R-PA-8 公测门 — 治理决策）。
3. **保留**应急后门 `KNOT_TOTP_BYPASS_ADMIN`（优先级 1）防唯一 admin 永久锁死。
4. 独立安全 PATCH v0.6.5.0（不绑 1.0；仍在 v0.6 MINOR 线内，角色不滚动）。

## §4 安全 / 锁死分析（守护者重点）

**删 bootstrap bypass 的锁死风险**：唯一 admin 弄丢 authenticator **且** 弄丢 10 个 recovery code → 无第二 admin reset → 锁死。
**缓解三层**：① 应急 env 后门 `KNOT_TOTP_BYPASS_ADMIN=true`（ops 逃生口）② enroll 时发 10 个 recovery code ③ enroll 流程白名单可达（强制 ≠ 锁死，admin 能完成 enroll）。
**OSS / demo 首次体验考量（守护者评）**：默认 on ⟹ `docker run` 评估者首登即被强制 enroll（需 authenticator）。缓解 = README/onboarding 明示 `KNOT_TOTP_REQUIRED=false` 关闭用于快速评估。资深已知悉并选择默认 on（安全优先）。
