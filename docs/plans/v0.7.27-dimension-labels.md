## 1. 根因 + 现状（grounded）

价值自测续（v0.7.23 follow-on D1）：语义层结果**维度列表头仍是英文列名**（`market` 而非 `交易对`、`sta_date` 而非 `日期`）。v0.7.23 只修了 **metric 列**表头（`column_labels = {metric.name → metric.display}`，走 payload），**维度列（`lf.dimensions`）原样透传英文**。

grounded 现状：
- **catalog 无任何列级中文**：`catalogs.tables` = `[{db, table, topics, summary}]`（列名埋在 `summary` 自由文本，非结构化）；`catalogs.lexicon` = `{业务词: [表全名]}`（**表级**，schema_filter/desensitize 用）。→ `market→交易对` 无现成来源，须建映射。
- **前端已就绪（关键简化）**：`TableContainer` 表头 `{columnLabels[c] || c}`（L54，对**所有列**生效）+ `MetricCard` labelCol/valueCol 同款（L29/35/51）→ **只要后端把维度标签 merge 进 `column_labels`，前端 0 改、0 新组件**。
- catalog 载体：`current_catalog()` 返 `{lexicon, tables, business_rules, relations, catalog_id}`；query 路径经 `query_helper._parse_catalog_content`（per-user active，ContextVar）；全局回退经 `catalog.py` 全局 + `catalog_loaders._load_from_db`（5-tuple）。

## 2. 决议（Option B · 资深 2026-07-02 拍板）

**catalog +`field_labels` JSON 键**（`{列名: 中文}`）—— 与 `lexicon` 同源同性（catalog-scoped 共享语义词汇），最轻（无新表/新屏/新端点，镜像 `relations` 加法），OOS-1 clean（catalog_id 拥有）。三方案对比见会话；A（独立表+屏）/D（DB COLUMN_COMMENT 自动抓）落选（重/不确定）。

**合并语义**（`_semantic_display_meta`）：metric 列走 `metric.display`（v0.7.23 sustained）；**维度列（`lf.dimensions`）补 `field_labels[d]`**。dims 与 metric.name 互斥（group-by 列 ≠ 聚合别名）→ 无冲突。`field_labels` 空/缺 → column_labels **byte-equal v0.7.25**（仅 metric 标签）。

**OOS**（本刀不做）：DB 列注释自动 introspection（D）· metric 列 unit 之外的 format · 非语义（LLM）路径维度标签（LLM 无 lf.dimensions，与 v0.7.23 D3 同 follow-on）。
