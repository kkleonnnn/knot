## §0 范围与方法 + grounding（最高守护 — grounded 实测）

`knot/services/agents/catalog.py` = **460 行**（收官①②未碰）· 0 `Optional[` · 有 `from __future__`（L16）。check_file_sizes ACK@:24（⏳ 待③移除）。

**头号风险 = facade-freeze 静默 bug（grounded）**：catalog.py 有 **5 个 mutable global**（`LEXICON`@24/`TABLES`@25/`BUSINESS_RULES`@26/`RELATIONS`@27/`_SOURCE`@28），被 **`reload()`**（L269-343，`global` 声明 L283，reassign L332/337/339/340/342）reassign，被 **10 个生产 importer 经 module-attr live-read**（`catalog_loader.LEXICON` 等）。
- **铁口（grounded 穷举）**：全仓 **ZERO 个 `from ...catalog import <global>` 值绑定**（latent 陷阱**未现形**）；10 importer 全 `from knot.services.agents import catalog as X` + `X.LEXICON`（module-attr）或经 `current_catalog()`（catalog.py 内部 L49-53 读 globals）。
- **拆分铁律**：5 globals + `reload()`（唯一 reassigner，持 `global`）+ `current_catalog()`（其 reader）**必须同物理 module** —— Python 名字绑定在定义模块 namespace，拆到别处则 reload 的 reassign 跨模块不可见（`global` 跨模块无效）。
- **若误用 package facade `from .state import LEXICON` re-export** → import-time 快照空 dict，reload 后 facade 名仍指旧空对象 → `catalog_loader.LEXICON` 静默读空 → 脱敏失效 / 选表全 0 分。**不会 CI 红**（reload 时序 pytest 未必触发）。

**关键约束（grounded — 颠覆「getter 全可抽」直觉）**：5 个 getter（`get_table_full_names`@363/`is_http_table`@390/`get_http_spec`@408/`get_field_mapping`@423/`get_http_tables`@434）**直读 global `TABLES`**（非 `current_catalog()`）；只 `get_relations`@374 + `get_relations_for_tables`@450 经 `current_catalog()`。抽 getter 出去 = 逼改 `current_catalog()`（**破 byte-equal 行为变更**，v0.6.2.6 D2 现状不动）或 `catalog.TABLES` module-attr（引 getter→catalog 循环边）。**故 getter 不抽**。

**circular（grounded）**：catalog.py 顶部 import = **纯 stdlib**（contextvars/importlib/json/pathlib，0 knot 内部）；所有 knot 依赖（repos/models.errors）**函数体内延迟 import** → import-time `reload()`@347 安全。拆分须**保持延迟 import 不变**（hoist 到子模块顶层会引 `catalog→repositories` import-time 环）。

## §1 决议（安全设计：只抽纯 loader · 回避 facade 陷阱整类）

> **核心判断**：目标（catalog ≤300 去 ACK）**仅靠抽 ~189 行纯 loader 即达成** → **根本不碰** state/getter/globals/facade 危险区。这比 grounding agent blueprint 的 package/getter-extraction **安全一个数量级**（后者引 facade-freeze + getter 循环 + 行为变更）。

**2-way 拆分（catalog.py 仍是 module 非 package）**：

| 文件 | 内容 | 行 | ≤300 |
|---|---|---|---|
| `catalog.py`（保留 module）| 5 globals(L24-28) + ContextVar+current_catalog+set/reset(L33-69) + **reload(L269-343) + 全部 getter(L350-460)** + 模块尾 auto-reload(L347) + import(改 +catalog_loaders) | 460−189 ≈ **272** | ✅ |
| `catalog_loaders.py` [NEW] | **4 纯 loader**：`_load_from_files`(L72-104) / `_load_from_db`(L107-169) / `_infer_source_types_from_datasources`(L172-237) / `_merge_lexicons`(L240-266) | ~189+header ≈ **200** | ✅ |

**为何安全（结构性回避守护者头号顾虑）**：
- 5 globals + reload + current_catalog + 全 getter **全留 catalog.py 同模块** → reload `global` reassign 同模块天然生效 → 10 importer module-attr live-read **不变** → **facade-freeze 整类不存在**（没有 package / 没有 facade / 没有 from-import re-export global）。
- loaders **纯计算返 tuple，0 global 读写，0 import catalog**（单向 `catalog → catalog_loaders`，无环）；reload() 调 `catalog_loaders._load_from_files()` 等，把返值经 `global` 塞 catalog.py 自己的 globals。
- **0 importer 改**（catalog 仍 module，X.LEXICON/X.reload() 全不变）· **0 行为变更**（getter 直读 TABLES 不动，不切 current_catalog）· **0 facade · 0 getter 循环**。
