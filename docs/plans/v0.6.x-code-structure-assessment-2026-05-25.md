## §0 范围与方法论

### 范围
- **代码主体**：v0.6.1.4 + 14 commit branch state（含未 merge）
- **指标**：LOC / 文件数 / 重复模式 grep / DEPRECATED 标记 / 测试比例
- **排除**：node_modules / __pycache__ / .pytest_cache / static build artifact / data/*.db

### 方法论
| 维度 | 工具 | 阈值 |
|---|---|---|
| 巨型文件 | `wc -l` 排行 | 单文件 > 350 LOC 标黄 / > 500 标红 |
| 重复 | `grep -l` 模式频次 | 同模式 ≥ 5 文件命中 = 抽象候选 |
| 死代码 | grep `DEPRECATED \| TODO \| FIXME` | 标记数 + target removal 版本 |
| 可读性 | 端点/职责密度 / 抽象层次缺失 | 单文件多职责 / 共享组件缺位 |

### 总览数据
```
后端 knot/   119 .py 文件   12,894 LOC
前端 src/    38 .jsx/.js   6,187 LOC
测试 tests/  76 .py 文件   8,683 LOC（比 67% — 偏高但优于偏低）
```
