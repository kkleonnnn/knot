## 1. 背景 + 根因（grounded · ResultBlock.jsx L87-89）

v0.7.23 图表硬化用 `dimension_cols=lf.dimensions`（语义路径权威）排除维度/ID 列，使 `numericCols` 只含真度量。但 **LLM 路径（非语义）+ 旧消息** `msg.dimension_cols` 空 → `dimensionCols=[]` → `numericCols = numericColsAll`（**全部数值列，含 user_id/持仓ID 等数值 ID 列**）→ ID 列被当 metric 画无意义图（#190）+ 进 MetricCard stat 网格。

## 2. 修（ID-like 名字启发式 · 仅 LLM fallback）

`isIdLikeCol(col)` 名字启发式 + LLM fallback 分支排除：
```js
const isIdLikeCol = (col) => /(^|_|[一-龥])id$|编号$/i.test(String(col));
const numericCols = dimensionCols.length
  ? numericColsAll.filter(c => !dimensionCols.includes(c))   // 语义路径权威（v0.7.23 不变）
  : numericColsAll.filter(c => !isIdLikeCol(c));             // D3：LLM 路径排除 ID-like
```

- **boundary 精准**（`^`/`_`/CJK 前 + `id$` 或 `编号$`，/i）：排除 user_id/order_id/id/持仓ID/账户ID/USER_ID/*编号；**保留** paid/void/bid/grid/rapid/gmv/dau/amount/count/pnl/fee/spot_amt（node regex 实测 7 排除全 true + 14 保留全 false）。
- **仅 LLM fallback**（`dimensionCols` 空时）：语义路径 `dimension_cols` 权威分支不走此（v0.7.23 byte-equal）。
- **non-ID 结果 byte-equal**：无 ID-like 列时 `filter` 0 剔除 = `numericColsAll`（旧行为，无回归）。
- **labelCols 不动**：ID-like 列仅从 `numericCols`（度量）排除，不加 labelCols → 若排除后无度量/无标签 → chartable=false → 表格 only（[user_id, gmv] → 表格；[city, user_id, gmv] → gmv by city）。
- **表格不受影响**：detail_table 用 `cols`（全列），ID 列仍在表中显示 —— 仅**图表 + MetricCard stat** 排除 ID 列。
