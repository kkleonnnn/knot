## §0 一句话 + ⚠️ 承重前提（R-137 自纠）

窗口函数解锁**排名**（每地区 GMV 前 3 城市）/**同环比**（GMV 环比）/**累计**（累计 GMV）—— 都是**窗口-over-聚合**（先 GROUP BY 算 metric，再对聚合结果开窗）。
**⚠️ 框法纠正（守护者 Stage 3 F1 — 我 + Stage 2 共写假前提）**：单层窗口+GROUP BY **合法**（标准 SQL 求值序 GROUP BY→**窗口**→SELECT，窗口在 GROUP BY 后求值，OVER 可引聚合表达式 `SUM(amt)`）；真限制是**同层 SELECT 不能引同层 alias**（OVER 须写 `SUM(amt)` 非 `gmv`，且多对象须在 OVER 内 `o.→t0` 重写）。
**两层是为 alias 干净而选**（外层 OVER 引子查询列 `gmv` = alias-based，免同层聚合表达式 + 多对象重写复杂），**非单层不可能**。仍比 HAVING 复杂（两层构造）。

→ 本草案核心 = 把窗口做成**外层包裹**（内层复用 v0.7.1~.8 已验编译 + 外层 OVER 引 alias 列），最小可行子集。

## §2 决策点（资深拍板项 · 执行者带推荐）

### D1 — ⭐ scope（推荐：窗口**列** via 外层子查询包裹，不做分区过滤）
- **✅ 推荐 MVP = 窗口列**：内层复用现编译（dims + metrics GROUP BY）→ 外层 `SELECT sub.*, <window_func> OVER (PARTITION BY ... ORDER BY ...) AS <name> FROM (内层) sub`。
  - 解锁**排名列**（`ROW_NUMBER()/RANK() OVER (PARTITION BY region ORDER BY gmv DESC) AS rank`）+ **同环比列**（`LAG(gmv) OVER (ORDER BY date)`）+ **累计列**（`SUM(gmv) OVER (ORDER BY date)`）—— 作为**附加列**呈现。
- **推迟（更复杂，后续刀）**：分区内 top-N **过滤**（`QUALIFY rank<=3` / 再包一层 WHERE）—— Doris QUALIFY 支持需核 + 三层构造；自定义 frame（ROWS/RANGE BETWEEN）。
- 理由：窗口列是两层最小构造（内层 0 改复用），交付 80% 价值（看排名/同比/累计）；分区过滤是三层，独立刀。

### D2 — LogicForm window 字段（推荐：结构化子对象）
- **✅ `window: [{func, partition_by, order_by, as_name}]`**（如 `{"func":"row_number","partition_by":["region"],"order_by":[{"field":"gmv","dir":"desc"}],"as_name":"rank"}`）。
- func 白名单：`row_number/rank/dense_rank`（排名）+ `lag/lead`（同环比）+ `sum/avg`（累计 OVER）—— **枚举非自由**（防注入 + 确定性）。⚠️ **func key 须映射真实 SQL 函数：`dense_rank` 非 `rank_dense`**（守护者 Stage 3 F3 — Stage 2 笔误；照抄生成不存在的 `RANK_DENSE()`）；`avg` 保留（Stage 2 误删）。
- order_by/partition_by 引 **metric alias 或 dimension**（alias-based，同 HAVING R-SL-80）。

### D3 — 编译两层构造（守护者 Stage 3 F2：inner 不调 _order_limit，非 string-strip）
- **✅ 有 window → 两层**：**inner = `sql + _having_clause`（不调 `_order_limit` → 本就无 LIMIT/ORDER BY，非生成后 string-strip）**；outer = `SELECT sub.*, <window> FROM (inner) sub` + **整个 `_order_limit`**（ORDER BY + LIMIT 都移外层；inner 不需 ORDER BY，窗口 OVER 自带排序，最终展示序在外层）。
- **无 window → 单层 byte-equal**（v0.7.1~.8 0 漂移，R-SL-84；走原 `sql + _having_clause + _order_limit` 不退化包 SELECT *）。

### D4 — 安全（推荐：func 枚举 + alias-based + _is_safe_sql）
- 窗口 func **枚举白名单**（非自由表达式）；partition_by/order_by 字段引 alias/dim（编译器构造 OVER，非 raw 透传）；最终 SQL 经 _is_safe_sql DQL-only 收口。
