### F2 — 时间语义统一引擎（v0.5 V4 ⭐⭐⭐ 必做）

#### F2.1 新建 `knot/core/time_resolver.py`（约 300 行）

提供单核函数：

```python
def resolve_time_context(
    natural_language_hint: str | None = None,
    today: date | None = None,           # 默认 datetime.date.today()
    data_freshness_lag_days: int = 1,    # 数据更新延迟（默认 D-1）
    timezone: str = "Asia/Shanghai",
) -> TimeContext:
    """统一时间语义解析。

    输入：
      - natural_language_hint: "今年" / "本月" / "上周" / "最近 7 天" / None
      - today: 显式指定（测试用）/ 默认今天
      - data_freshness_lag_days: 数据更新延迟（默认 D-1 即昨天）

    输出 TimeContext:
      - today_str: '2026-05-14'
      - latest_data_date_str: '2026-05-13'（today - lag）
      - this_year: ('2026-01-01', '2026-12-31')
      - this_year_to_latest: ('2026-01-01', '2026-05-13')  ← 同比基准
      - this_month: ('2026-05-01', '2026-05-31')
      - this_month_to_latest: ('2026-05-01', '2026-05-13')
      - last_week: ('2026-05-04', '2026-05-10')
      - last_7_days_to_latest: ('2026-05-07', '2026-05-13')
      - same_period_last_year: ('2025-05-07', '2025-05-13')  ← 同比对照
      - holidays_in_range: ['2026-05-01' (劳动节)]  ← 节假日提示
      - is_holiday_today: False
      - prompt_block: str  ← 注入 SqlPlanner system prompt 的标准段
    """
```

#### F2.2 内置规则

1. **节假日表**（2026 年）：
   - 元旦：2026-01-01 ~ 01-03
   - 春节：2026-02-17 ~ 02-23
   - 清明：2026-04-04 ~ 04-06
   - 劳动节：2026-05-01 ~ 05-05
   - 端午：2026-06-19 ~ 06-21
   - 中秋：2026-09-25 ~ 09-27
   - 国庆：2026-10-01 ~ 10-08
   - hardcoded 字面 dict（v0.7+ 可升级为外部 yaml / admin 可编辑）

2. **数据更新延迟**：
   - 默认 D-1（昨天数据）
   - admin 可配置（app_settings KV `time_resolver_lag_days`）
   - 影响"今年到目前"/"截至最新"等表达

3. **周/月/季对齐**：
   - 周一为周首（ISO 8601）
   - 月：自然月（2026-05-01 ~ 2026-05-31）
   - 季：自然季（Q2 = 2026-04-01 ~ 2026-06-30）

#### F2.3 升级 `knot/core/date_context.py`

当前 `date_context.py` 仅注入"今天日期"字面。升级为：
- 调用 `time_resolver.resolve_time_context()`
- 返回 `TimeContext.prompt_block`（含完整时间语义注入）

prompt_block 示例：
```
## 时间语义上下文（KNOT 时间引擎 v0.6.1）

- 今天：2026-05-14（周四）
- 最新数据：2026-05-13（D-1，数据更新延迟 1 天）

- "今年" = 2026-01-01 至 2026-12-31（全年范围）
- "今年到目前" / "截至最新" = 2026-01-01 至 2026-05-13
- "本月" = 2026-05-01 至 2026-05-31
- "本月到目前" = 2026-05-01 至 2026-05-13
- "上周" = 2026-05-04 至 2026-05-10（ISO 周一首）
- "最近 7 天" = 2026-05-07 至 2026-05-13

- "同比" 基准（vs 去年同期）：
  - "今年同比" = 2025-01-01 至 2025-05-13（与"今年到目前"对照）
  - "本月同比" = 2025-05-01 至 2025-05-13

- 节假日提示：本月含劳动节假期（2026-05-01~05-05）

⚠️ 用户问"今年"/"本月" 默认按"到目前"截断（不含未来）— 除非用户明示"全年"
```
