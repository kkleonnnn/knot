## 3. 技术选型

### 3.1 表结构：固定列 + JSON 详情列（混合）

```sql
CREATE TABLE IF NOT EXISTS audit_log (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    actor_id      INTEGER,        -- NULL 仅用于失败登录（无身份）
    actor_role    TEXT,            -- 'admin' / 'analyst' / NULL（匿名失败登录）
    actor_name    TEXT,            -- 冗余 username 快照（即使用户后续被删审计仍可读）
    action        TEXT NOT NULL,   -- 见 §3.2 enum
    resource_type TEXT NOT NULL,   -- 'user' / 'datasource' / 'budget' / 'api_key' / ...
    resource_id   TEXT,            -- 主键值（int 或 string）字符串化
    success       INTEGER DEFAULT 1, -- 0/1，失败也记（如登录失败）
    detail_json   TEXT DEFAULT '', -- 结构化变更细节 / 失败原因 / IP / UA
    ip_address    TEXT,            -- 来自 X-Forwarded-For 或 client.host
    request_id    TEXT,            -- 串 v0.2.2 logger middleware 注入的 request_id
    created_at    TEXT DEFAULT (datetime('now','localtime'))
);

CREATE INDEX idx_audit_actor_time ON audit_log(actor_id, created_at);
CREATE INDEX idx_audit_action_time ON audit_log(action, created_at);
CREATE INDEX idx_audit_resource ON audit_log(resource_type, resource_id);
```

理由：
- 固定列（actor / action / resource）便于 SQL WHERE 筛选 + 索引
- `detail_json` 灵活承载 diff / IP / UA / 失败原因
- `actor_name` 冗余 — 用户被 `is_active=0` 后审计仍可读
- INSERT-only（不暴露 UPDATE / DELETE 路由 — 仅 purge 脚本可批量删过期）

### 3.3 写入策略：同步 INSERT（不异步）

**决策**：单实例 SQLite，INSERT 单事务 ~1ms，无需 BackgroundTasks 引入复杂性。
- 同步写入；`audit_service.log()` 内部 try/except — 写入失败 logger.error 但**不阻断业务**（R-47）
- 失败原因（如 audit 表 schema mismatch）**不能**让登录失败

### 3.4 Retention（90 天默认）

- `app_settings.audit_retention_days` 默认 `90`
- admin 可调（路由 PUT `/api/admin/audit-config`）
- 调整本身入审计（meta-audit `audit.retention_change`）
- Purge 脚本：`bi_agent/scripts/purge_audit_log.py`（独立 entrypoint，参 v0.4.5 migrate 模式）
- **不**接 startup hook 自动 purge（避免每次重启删数据；admin 主动跑 / cron 触发）

## 4. API 设计

### 4.1 `bi_agent/services/audit_service.py`（NEW）

```python
"""审计写入入口（v0.4.6）。

红线：
- R-47 失败不阻断业务（fire-and-log）
- R-48 PII 严禁入 detail_json（密码 / API Key 明文 → 必须 mask）
- R-51 actor 从 token 取（不能信 client）
- R-55 action 必走 AuditAction Literal
"""
from __future__ import annotations
from typing import Any
from bi_agent.models.audit import AuditAction, AuditResourceType
from bi_agent.repositories import audit_repo
from bi_agent.core.logging_setup import logger


# 不入 detail_json 的字段名（PII 防御）
_PII_BLACKLIST = frozenset({
    "password", "password_hash", "api_key", "openrouter_api_key",
    "embedding_api_key", "doris_password", "db_password",
})


def _scrub_pii(d: dict | None) -> dict:
    """递归扫描 dict，命中 _PII_BLACKLIST 的 key 替换为 '••••redacted••••'。"""
    if not d:
        return {}
    out = {}
    for k, v in d.items():
        if k in _PII_BLACKLIST:
            out[k] = "••••redacted••••"
        elif isinstance(v, dict):
            out[k] = _scrub_pii(v)
        else:
            out[k] = v
    return out


def log(
    *,
    actor: dict | None,        # 当前 user dict（None = 匿名失败登录）
    action: AuditAction,
    resource_type: AuditResourceType,
    resource_id: str | int | None = None,
    success: bool = True,
    detail: dict | None = None,
    ip: str | None = None,
    request_id: str | None = None,
) -> None:
    """fire-and-log：业务路径必须用 try-free 调用（R-47）。"""
    try:
        audit_repo.insert(
            actor_id=actor["id"] if actor else None,
            actor_role=actor["role"] if actor else None,
            actor_name=actor["username"] if actor else None,
            action=action,
            resource_type=resource_type,
            resource_id=str(resource_id) if resource_id is not None else None,
            success=1 if success else 0,
            detail_json=_scrub_pii(detail) if detail else {},
            ip_address=ip,
            request_id=request_id,
        )
    except Exception as e:
        logger.error(f"[audit] 写入失败（业务不阻断）: {e}; action={action}")
```

### 4.3 路由集成（关键 8 类 mutation）

每个 mutation 路由结束时调用 `audit_service.log(...)`。**注入位置在 api 层**（4 层契约：services 不该知道审计细节）。

抽 helper 减重复：

```python
# bi_agent/api/_audit.py（NEW）
from fastapi import Request
from bi_agent.services import audit_service

def audit(request: Request, actor: dict | None, **kwargs):
    """API 层 helper：自动从 request 取 IP / request_id。"""
    audit_service.log(
        actor=actor,
        ip=request.client.host if request.client else None,
        request_id=request.headers.get("x-request-id"),
        **kwargs,
    )
```

### 4.5 GET / PUT `/api/admin/audit-config`（NEW）

```python
@router.get("/api/admin/audit-config")
async def get_audit_config(admin=Depends(require_admin)):
    return {"retention_days": int(settings_repo.get_app_setting("audit_retention_days", "90"))}

@router.put("/api/admin/audit-config")
async def set_audit_config(payload: dict, request: Request, admin=Depends(require_admin)):
    new_days = int(payload.get("retention_days", 90))
    if not (7 <= new_days <= 3650):
        raise HTTPException(400, "retention_days 必须在 7~3650 区间")
    old = settings_repo.get_app_setting("audit_retention_days", "90")
    settings_repo.set_app_setting("audit_retention_days", str(new_days))
    audit(request, admin, action="audit.retention_change", resource_type="audit",
          detail={"from": old, "to": new_days})
    return {"ok": True}
```

### 4.6 Purge 脚本

```python
# bi_agent/scripts/purge_audit_log.py
"""审计 retention 清理（独立 entrypoint）。

执行：python3 -m bi_agent.scripts.purge_audit_log [--dry-run]
"""
def purge(dry_run: bool = False) -> dict:
    days = int(settings_repo.get_app_setting("audit_retention_days", "90"))
    deleted = audit_repo.delete_older_than(days, dry_run=dry_run)
    if not dry_run and deleted > 0:
        # meta-audit：purge 自身入审计（R-57）
        audit_service.log(
            actor=None, action="audit.purge", resource_type="audit",
            detail={"days": days, "deleted_count": deleted},
        )
    return {"days": days, "deleted": deleted}
```

## 9. 决策待定项（守护者明确）

| ID | 决策点 | 倾向 | 备选 |
|---|---|---|---|
| **D1** | 同步 vs 异步 INSERT | **同步**（单实例 SQLite ~1ms） | 异步 BackgroundTasks（v0.5+ 集群再考虑） |
| **D2** | retention purge 自动化 | **手动 / cron 触发**（不进 startup hook） | startup lazy purge / 定期 background task |
| **D3** | 前端 AdminAudit 是否本 PATCH 落 | **必备**（log 沉睡 = 没审计） | 推 v0.4.7（守护者可决） |
| **D4** | message_id 是否入 audit | **不入**（已用 messages 表覆盖；audit 通过 resource_id=message_id 关联） | 入（重复登记） |
| **D5** | failed login 是否记 actor_name | **记**（即使 username 不存在；便于发现 password spray 攻击） | 不记（保护未注册用户身份） |
| **D6** | retention 区间下限 | **7 天**（防误调成 0 天清空所有） | 1 天 / 30 天 |
| **D7** | audit_log 是否加密 detail_json | **否**（不存 PII；明文便于 admin 直接 sqlite3 查） | 加密（与 v0.4.5 同模式） |

## 10. 兼容矩阵 / 风险点

| 场景 | 行为 | 风险 |
|---|---|---|
| v0.4.5 老 DB 升级 | `init_db()` 自动建 `audit_log` 表 + 索引；老 mutation 无 audit 记录（合理） | 接受 — 升级前操作不可追溯，自然边界 |
| audit_repo 写入失败 | logger.error + 业务不中断（R-47） | 漏审 — 但保业务 SLA |
| 失败登录密度高 | audit_log 增长快；retention 90 天可能压力 | 90 天默认仅 1k rows/天预估，无压力；万级集群推 v0.5+ |
| admin 改 retention 到 7 天再切回 90 | 已 purge 数据找不回 | 接受 — meta-audit 留 trace |
| meta-audit 死循环 | audit.purge 入 log → 下次 purge 又删它 | retention 区间内自然过期 |
| PII 误入 detail | `_PII_BLACKLIST` 漏字段 | 加 CLAUDE.md 流程红线「新增 sensitive field 必须更新 _PII_BLACKLIST」 |
