## 1. 决议（窄 · 编译覆盖续末个窄候选）

**一句话**：让窗口函数支持自定义 `ROWS BETWEEN <start> AND <end>` frame —— 解锁 **N 日移动平均/滑动窗口/滑动和**（如 7 日移动平均 `AVG(gmv) OVER (ORDER BY date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW)`）。

### ⭐ 核心判断（grounded main `786c557`）

- v0.7.9 窗口（`_window_col`）只有 `OVER (PARTITION BY .. ORDER BY ..)` —— 默认 frame（有 ORDER BY 时 = `RANGE UNBOUNDED PRECEDING AND CURRENT ROW`，即累计）。**自定义 frame 解锁滑动窗口**（移动平均/滑动和），是真实趋势平滑高频分析。
- **frame 是 window dict 子字段**（`window: [{func, arg, partition_by, order_by, as_name, frame}]`）→ **logicform 0 改**（window canonical = `dict(sorted(w.items()))` 自动纳入 frame key；SQL 经 `_frame_clause` 显式读 preceding/following 保 determinism，与 window order_by 同处理）。
- **⚠️ frame 是 SQL 注入面**（`ROWS BETWEEN <值>`）→ 边界**结构化 + 严格白名单**（仅非负 int 或 `"unbounded"` 枚举，0 用户字符串裸拼）。
- **frame 仅对聚合窗口（sum/avg）有意义** → ranking（row_number/rank/dense_rank）/ lag/lead + frame → raise；frame 需 ORDER BY（无 → raise）。

```sql
-- 「GMV 7 日移动平均」
SELECT sub.*, AVG(gmv) OVER (ORDER BY date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS ma7 FROM (...) sub
```

### In-Scope
- compile_helpers `_frame_clause` + `_frame_bound`（注入安全 SQL 片段，同 `_order_limit` leaf 定位）
- compiler `_window_col` frame gate + append
- parser prompt 教 frame

### OOS
- `RANGE BETWEEN` / `GROUPS BETWEEN` frame 模式（仅 ROWS，最常用）
- 派生指标模型（metric÷metric）/ 主动监控深化

## 2. 设计

### 2.1 frame 字段（window dict 子字段 · logicform 0 改）

window dict +`frame`：`{"frame": {"preceding": <int≥0 | "unbounded">, "following": <int≥0 | "unbounded">}}`。
- 7 日移动平均：`{"func":"avg","arg":"gmv","order_by":[{"field":"date","dir":"asc"}],"frame":{"preceding":6,"following":0},"as_name":"ma7"}`。
- logicform **0 改**：window 已 `list[dict]`，frame 是子 key；canonical `dict(sorted(w.items()))` 自动纳入（R-SL-125）。

### 2.2 compile_helpers `_frame_clause` + `_frame_bound`（注入安全）

```python
def _frame_clause(w: dict, takes_arg: bool, has_order_by: bool) -> str:
    """ROWS BETWEEN frame（v0.7.15）：gate 仅 sum/avg 聚合窗口 + 需 ORDER BY；边界防注入。"""
    if not takes_arg or str(w.get("func")) in ("lag", "lead"):
        raise CompileError(f"frame 仅 sum/avg 聚合窗口支持（func={w.get('func')!r}）→ 回退")
    if not has_order_by:
        raise CompileError("frame（ROWS BETWEEN）需 ORDER BY → 回退")
    f = w["frame"]
    return f"ROWS BETWEEN {_frame_bound(f.get('preceding'), 'PRECEDING')} AND {_frame_bound(f.get('following'), 'FOLLOWING')}"


def _frame_bound(v, side: str) -> str:
    """frame 边界（防注入：仅非负 int 或 'unbounded'；0 → CURRENT ROW）。"""
    if isinstance(v, str) and v.strip().lower() == "unbounded":
        return f"UNBOUNDED {side}"
    if isinstance(v, bool) or not isinstance(v, int) or v < 0:   # 排 bool + 仅非负 int（0 字符串裸拼）
        raise CompileError(f"frame 边界 {v!r} 须非负 int 或 'unbounded' → 回退")
    return "CURRENT ROW" if v == 0 else f"{v} {side}"
```

**注入安全**：边界仅 `非负 int`（`isinstance int` + 排 bool + `≥0`）或 `"unbounded"` 枚举；输出全受控（int repr + 字面 PRECEDING/FOLLOWING/CURRENT ROW/UNBOUNDED），**0 用户字符串裸拼**。非法 → raise 回退。

### 2.3 compiler `_window_col` frame 集成

```python
    if obs:
        over.append("ORDER BY " + ", ".join(obs))
    if w.get("frame"):
        over.append(_frame_clause(w, takes_arg, bool(obs)))   # gate + ROWS BETWEEN（compile_helpers）
    return f"{sqlfunc}({arg}) OVER ({' '.join(over)}) AS {w.get('as_name') or w.get('func')}"
```
**无 frame → byte-equal**（`if w.get("frame")` 不触 → v0.7.9 两层 SQL 不变 R-SL-128）。

### 2.4 parser

`_LOGICFORM_SYS` window 教学加 frame：
> window 可选 `frame`（**仅 sum/avg 滑动窗口/移动平均**，如「GMV 7 日移动平均」→ func=avg+order_by=[date asc]+frame={{"preceding":6,"following":0}}）：`preceding`/`following` 取**非负整数**（0=当前行）或 `"unbounded"`；**ranking/lag/lead 不支持 frame**。无滑动需求 → 省略 frame。
