> **简化协议**（资深 2026-07-02 拍「小批 robustness → b merge only，type-aware defer YAGNI」；小 defensive 逻辑修 + no-collision 路径 byte-equal + grounded 无争议）。非阻塞 follow-on 池「小批 robustness」拆分后单项。**0 新 R-SL**。

## 1. 背景 + 根因（grounded）

catalog `reload()` 合并 DB catalog.tables（admin 编辑）+ file catalog.tables（`_local`/`_template_catalog.py` 部署代码层）。**HTTP 虚拟表是部署代码层配置**（catalog.py L80-82 架构铁律：「HTTP 表不应被 admin DB 编辑覆盖」）。

**bug（catalog.py:113-122 旧逻辑）**：`for t in http_from_file: if full not in existing_names: append` —— file http 表仅在**名不在 DB** 时追加。若 admin 手灌同名条目进 DB catalog.tables（「01 表目录」编辑器只有 {db,table,topics,summary}，**无 source_type** 字段），该手灌影子 `is_http_table()=False` → `pick_http_route` 漏 → **静默落 SQL**（= problem 1 生产 bug 类，e38de5e76703）。DB 影子**遮蔽**权威 file http。

## 2. 修（file http 权威覆盖）

`reload()` `if db_tables:` 块：先剔除 base_tables 中与 file http **同名**的条目（影子），再追加全部 file http（权威）。
```python
http_names = {f"{db}.{table}" for t in http_from_file}
base_tables = [t for t in base_tables if f"{db}.{table}" not in http_names]  # 剔影子
base_tables.extend(http_from_file)                                           # 追权威
```

- **no-collision byte-equal**：DB SQL 表不与 file http 同名 → 过滤 0 剔除 + 追加同旧（正常路径不变）。
- **collision（修）**：DB 手灌影子被剔 + file http（source_type=http）追加 → `is_http_table`=True → pick_http_route 命中。
- `db_tables` 空路径（file-only）不进本块 → 不变。`_infer_source_types_from_datasources`（L131）对已带 source_type=http 的 file 表幂等。
