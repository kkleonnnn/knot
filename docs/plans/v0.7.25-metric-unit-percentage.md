# v0.7.25 — metric `unit` 字段 + 比率百分比渲染（D2 · #1 比率% · Stage 1 草案）

> **触发**：价值自测 #1。派生费率指标（future_fee_rate 等 4 个 divide 派生）卡片显「0」——grounded：值 = 0.000486，
> MetricCard `_fmt` 用 `toLocaleString()`（默认 maximumFractionDigits=**3**）→ 0.000486 舍入 3 位 = 0.000 = **「0」**（格式非值）。
> **资深拍 scope**：unit 字段通用 + **仅 percentage 渲染**（最小解「0」；currency 等 unit 支持存储但暂不特殊渲染）+
> MetricCard+表格（图表原值 defer）+ admin UI include + **content 派生补录单独**（kk/我，用新 unit 字段）。
> **协议**：完整 v3。**D2 = v0.7.25**（Layer C silent-wrong 顺延 v0.7.26+）。

## 1. 根因 + 现状（grounded）

- **「0」= toLocaleString 3 位舍入**：`MetricCard.jsx` `_fmt(value) = typeof number ? value.toLocaleString() : String(value)`；`(0.000486).toLocaleString()` 默认 maximumFractionDigits=3 → 0.000 →「0」。（表格 cell `String(row[c])` 显 0.000486 全，但卡片大数字舍成 0。）
- **已有 4 费率派生 metric**（全 divide、全 ratio、全会显「0」）：`net_deposit_rate 净充率` / `withdraw_rate 提现率` / `future_fee_rate 合约费率` / `spot_fee_rate 现货费率`（`lineage={"op":"divide",...}` caliber 空）。
- **metric 模型无 unit/format 字段**（`models/metric.py`：name/display/aliases/caliber/base_object/filters/dimensions/date_column/lineage/freshness_lag_days/enabled）→ 需加（镜像 v0.7.17 `date_column`）。
- **呈现 meta 机制已在**（v0.7.23）：`_semantic_display_meta(lf, by_name)` 返 `(column_labels, dimension_cols)` → AgentResult → payload → 前端。D2 **扩** column_formats。

## 2. 决议（unit 字段 + percentage 渲染 · 扩 v0.7.23 呈现 meta）

- **后端 metric `unit` 字段**（镜像 v0.7.17 date_column 全套）：`models/metric.py` +`unit: str = ""`；`metric_repo` `_COLS`+unit / `_UPDATABLE`+unit / 幂等 `ALTER TABLE metrics ADD COLUMN unit`（镜像 date_column ALTER）；create/update endpoint 经 `_UPDATABLE` 自然流（additive）。默认 `""` → 现有 metric byte-equal。
- **呈现 meta 扩 column_formats**：`_semantic_display_meta` 返 **3-tuple** `(column_labels, dimension_cols, column_formats)`；`column_formats = {n: by_name[n].get("unit") or "" for n in lf.metrics if n in by_name and (by_name[n].get("unit") or "")}`（仅非空 unit 入）；`sql_planner.AgentResult` +`column_formats: dict = {}`（additive；LLM/sync byte-equal）；`query.py` payload +`column_formats`；`Chat.jsx` onFinal +`column_formats: ev.column_formats || {}`。
- **前端 percentage 渲染**：共享 helper `fmtValue(value, unit)` 放 **`chat/ResultBlock/fmt.js`**（新 tiny 模块，MetricCard + TableContainer 两处 import 避重复 · 守护者 R3）——**非-percentage 路径完整 subsume 现 `_fmt` 逻辑 byte-equal**（`null/undefined→'—'` + `number→toLocaleString()` + `String`，MetricCard `_fmt`:10-12 三元逐字）；**percentage + typeof number → `(value*100).toLocaleString(undefined, {maximumFractionDigits: 4}) + '%'`**。MetricCard（`_SingleMetric` 大数字 + `_MultiStatGrid` stat）+ TableContainer cell 用 `fmtValue(v, columnFormats[col])`。**unit 空 / 非 percentage → byte-equal 现状**。
  - ⚠️ **R1 承重约定（守护者 Stage 3）**：percentage 假设**值是 0-1 小数 fraction**（÷派生费率如 fee/volume=0.000486 → ×100=0.0486% ✓）。若设在 caliber 已返百分数（如 `AVG(rate)*100`=4.86）的 metric 上 → ×100=**486% 静默双缩放**。→ **约定：unit=percentage 的 metric 值必须是 0-1 小数**（content 补录 4 费率 + 未来 percentage metric 都须遵循）；admin UI hint 写明（R-SL-184）。
  - ⚠️ **R2 note（守护者）**：`maxFractionDigits:4` 是舍入**移位非根除**——realistic 费率范围（0.01%~几%）足够，sub-0.0001% 极小费率 ×100 后仍可能舍向「0%」（残留边界，可接受）。
- **admin UI**：`AdminMetricRegistry` +unit field（下拉 percentage / 空；镜像 date_column field）+ **percentage 选项 hint「百分比 — 值为 0-1 小数，显示自动 ×100%」**（R1 承重 · 防设在已 ×100 口径上双缩放）。
- **图表 defer**：ResultBlock chartData 原值不动（% 轴格式复杂 → follow-on）。

### Defer / content（本刀不做代码）

- **content 派生补录**（kk/我）：4 个已有费率 metric 设 unit=percentage + 注册新派生（资金费率/返佣费率/客单价…）。代码 PATCH 落地后单独补（本地 DB + 导出运维）。⚠️ **R1 前置**：设 unit=percentage 前**务必确认 caliber 返 0-1 小数 fraction**（4 费率是 divide 派生 fee/volume ✓）；**严禁**设在已 ×100 的口径上（双缩放）。客单价/arpu 非 fraction（是金额）→ **不设 percentage**（unit 空或 currency）。
- **currency 等 unit 渲染**：unit 字段支持存储，但暂不特殊渲染（toLocaleString 默认；客单价等大数无「0」问题）→ follow-on。
