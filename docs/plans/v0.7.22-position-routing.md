> **触发**：kk 价值自测第二轮 #3「本月各交易对的持仓盈亏」/ #4「各交易对持仓盈亏排名」→「持仓」命中 HTTP

> 机制从「时间窗」校正为「**intent**」—— grounded：clarifier 产出无结构化时间字段、time_resolver 不解析文本 →

## 1. 根因（grounded）

- `pick_http_route(refined_question)` 在 clarifier 之后、语义层之前跑（[`query.py:248`](../../knot/api/query.py)）；只拿 `refined_question` 字符串。
- Layer 3 `_EXCLUSION_RE`（[`http_planner.py:97`](../../knot/services/http_planner.py)）排「历史/已平仓/强平/爆仓/ADL/N天前/N月前/昨天/前天/上周/上月」——**漏「本月/各/排名/盈亏」**。
- 「持仓」lexicon → `futures_position_list`（HTTP 当前快照，market+side 必填）→ **抢路由** → #3/#4 **到不了语义层** `user_position_pnl`。
- **物理事实**：HTTP futures 快照只能答 `metric`（count/total）或 `detail`（list）；**结构上产不出** `trend/compare/rank/distribution/retention`（需历史/聚合 SQL）。
- **clarifier 产出 grounded**：`{is_clear, refined_question, analysis_approach, intent}` —— 无时间字段；`intent ∈ VALID_INTENTS` 7 类（metric/trend/compare/rank/distribution/retention/detail）是**唯一现成结构信号**。
- **clarifier.md point 7（v0.6.1.12）已为 futures 持仓定分类规则**：默认 detail / 总量→metric / 排序→rank / 分布→distribution（Layer B 在此基础上微调，非新建）。

## 2. 决议（Layer A + B 配对；Layer C defer）

- **Layer A（核心 · intent 结构信号）**：`pick_http_route` 收 `intent` 参数（additive 2nd param，默认 `None`）。
  **`intent ∈ {trend, compare, rank, distribution, retention}`（5 分析类）→ `return None`（走 SQL）**——HTTP 快照产不出这 5 类。
  ⚠️ **位置（守护者 R2 · 第 5 锚点 · 真 correctness）**：Layer A 必须 **early-return 在 Layer 1 关键词匹配之前**（函数体首条 `if intent in {5类}: return None`，**在 `_EXCLUSION_RE` 与 lexicon loop 之前**）——否则「持仓」lexicon 先命中返 HTTP route，intent veto 永不执行（同守护者 v0.7.16「dispatch 先于 `_metric_bases`」class）。
  确定性（intent 是 clarifier 既有分类，非措辞匹配）。→ **解 #4（排名→rank→SQL）**。
  📝 **隐含假设（守护者 nit · 登记）**：Layer A 设「HTTP=快照-only，产不出分析类」——对**当前** futures catalog 成立；但 HTTP adapter 是**通用 catalog 驱动**，若未来加分析型 HTTP 端点（如 `/kline` 历史时序=trend）→ Layer A 会把其分析 intent 误路由 SQL。**登记此假设**，HTTP catalog 长出分析端点时重访（当前对，不阻断）。
- **Layer B（clarifier.md point 7 微调 · Layer A 正确性承重）**：「**各X/按X 的盈亏/金额**」聚合语 → `rank/distribution`（非 detail）→ Layer A 接 → **解 #3**。
  ⚠️ **边界（守护者承重 check）**：「**当前/实时持仓**」裸列表**仍归 detail** → Layer A 不接 detail → **HTTP 正路不断**。Layer B 只调「有 各/按 + 盈亏/金额 聚合语」，不动裸快照。
- **Layer 3 `_EXCLUSION_RE` 保持 byte-equal 不扩**（守护者 anti-whack-a-mole）——它是 detail+历史词的现有部分覆盖，不删不扩。
- **Layer C 残留（defer v0.7.25 · 登记 silent-wrong 真 bug）**：单交易对+时间窗（「本月BTC盈亏」intent=metric）→ Layer A/B 都不接 → 仍漏 HTTP。
  = **silent-wrong bug**（HTTP 忽略时间窗返当前快照、冒充本月、**不报错** — 比报错更隐蔽）。metric intent 对「快照-vs-时间窗」**本质歧义**（「当前BTC持仓量」HTTP 对 / 「本月BTC盈亏」SQL 对，俩都 intent=metric）。
  真需 **time-window 文本检测器**（独立子工程；regex 是打地鼠；**语义层时间窗复用 = 复合价值正经做**）。**无自测 case 命中** → v0.7.22 defer 可接受；CHANGELOG + 本 plan **登记真 bug 优先级非 robustness polish**。
- **依赖（content，已落）**：Layer A 路由到 SQL 后，#3/#4 **成功**依赖 `user_position_pnl` metric（problem 2 已 seed aliases「各交易对盈亏」等 + dwd 预过滤）被语义 parser 命中 / sql_planner 找到 dwd 表。本刀只修路由，不动 content。
